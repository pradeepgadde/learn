Stop using os.path. Use pathlib.

Paths are objects, not strings. Once that clicks, half your file-handling code disappears.

02 Jan 2026 • ~3 min read • Python 3.6+
TL;DR

pathlib.Path is the modern way to deal with files. Use / to join, .read_text() to read, .glob() to search, and never type os.path.join again.

os.path was designed when paths were just strings. That's why every operation is a free-standing function: os.path.join, os.path.exists, os.path.dirname. pathlib, added in Python 3.4, treats paths as what they actually are — objects with methods. The result is code that reads like a sentence.

Joining paths with the / operator

The / operator is overloaded for Path objects. It joins paths with the correct separator on every OS, no string concatenation, no os.sep, no surprises on Windows. 

pythonfrom pathlib import Path

project = Path("~/code/myapp").expanduser()
config  = project / "settings" / "prod.toml"

print(config)             # /home/pradeep/code/myapp/settings/prod.toml
print(config.exists())

Reading and writing in one line

For small files, you almost never need open() + a with-block any more. 

pythonnotes = Path("notes.md")

notes.write_text("# Day 002\n\nPathlib is great.")
contents = notes.read_text()

# Bytes versions exist too:
data = Path("image.png").read_bytes()

Globbing instead of listdir + filter

glob() walks one directory level. rglob() walks recursively. Both return generators of Path objects, ready to filter or iterate. 

pythonrepo = Path(".")

# All Python files at any depth, skipping virtualenvs
py_files = [
    p for p in repo.rglob("*.py")
    if "venv" not in p.parts
]
print(f"{len(py_files)} Python files")

The properties you'll use constantly

These are the four you'll reach for over and over — and they don't require allocating any new strings. 

pythonp = Path("/var/log/nginx/access.log.gz")

p.name     # 'access.log.gz'
p.stem     # 'access.log'
p.suffix   # '.gz'
p.suffixes # ['.log', '.gz']
p.parent   # PosixPath('/var/log/nginx')
p.parts    # ('/', 'var', 'log', 'nginx', 'access.log.gz')

Need to swap an extension? p.with_suffix(".txt"). Need a sibling file in the same directory? p.with_name("other.log"). Both return new Path objects without touching the filesystem.

Open any script in your codebase that imports os.path. Replace the imports and rewrite the path handling using pathlib. Count the lines you deleted — usually 20–40%.

Bonus: find a place where you wrote open(path).read() and shorten it to Path(path).read_text().
