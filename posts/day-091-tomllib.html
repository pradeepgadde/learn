tomllib: TOML config in the stdlib

Python 3.11 added tomllib for reading TOML. Clean typed config with comments and multiline strings — no extra dependencies.

1 Apr 2026 • ~2 min read • Python 3.11+
TL;DR

import tomllib (3.11+) or pip install tomli for older Pythons. Open in binary mode, call tomllib.load(f). Get back a plain dict. TOML is the format of pyproject.toml and a clean alternative to YAML for config files.

TOML (Tom's Obvious Minimal Language) is what pyproject.toml, Cargo, and many modern tools use for configuration. It's human-readable, strongly typed, and has no YAML-style indentation ambiguity. Since 3.11 it's in the standard library.

Reading a TOML file

toml# config.toml
[app]
name    = "poller"
version = "1.2.0"
debug   = false

[database]
host     = "db.internal"
port     = 5432
timeout  = 30.0

[devices]
roles  = ["access", "core", "distribution"]
prefix = "10.0.0.0/8"
pythonimport tomllib

with open("config.toml", "rb") as f:   # binary mode required
    cfg = tomllib.load(f)

cfg["app"]["name"]        # 'poller'   (str)
cfg["database"]["port"]    # 5432       (int)
cfg["database"]["timeout"] # 30.0      (float)
cfg["app"]["debug"]        # False     (bool)
cfg["devices"]["roles"]    # ['access', 'core', 'distribution'] (list)

Types come back exactly as written. No "5432" string that you forget to cast; no boolean-as-string traps.

From a string (for tests)

pythonraw = b"""
[server]
host = "localhost"
port = 8080
"""
cfg = tomllib.loads(raw.decode())      # loads() takes str, not bytes

Pair with pydantic for validated config

pythonfrom pydantic import BaseModel
import tomllib

class DBConfig(BaseModel):
    host:    str
    port:    int = 5432
    timeout: float = 30.0

class Config(BaseModel):
    database: DBConfig

with open("config.toml", "rb") as f:
    cfg = Config.model_validate(tomllib.load(f))

TOML vs YAML vs JSON for config

  • TOML — typed, comments, multiline strings, arrays of tables. Best for human-edited config files with hierarchy. The standard for Python tooling.
  • YAML — more expressive but infamous for surprise: no becomes False, 1e3 becomes 1000.0. Good for k8s manifests because the ecosystem already uses it.
  • JSON — no comments, no trailing commas. Fine for machine-generated config; poor for human-edited.

Python < 3.11: tomli

pythontry:
    import tomllib
except ImportError:
    import tomli as tomllib   # pip install tomli

tomli is the backport; identical API, same binary-mode requirement.

tomllib is read-only by design. For writing TOML, use tomli-w (pip install tomli-w). Keeping read and write separate keeps the stdlib module small and dependency-free.

Replace one JSON or YAML config file in a project with TOML. Move your json.load to tomllib.load. Enjoy the comments you can now write in the config, and the integers that stop being strings.
