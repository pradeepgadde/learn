Counting anything with Counter

A dict that tallies, with a top-N method and arithmetic between counters. The cleanest tool in the standard library.

03 Jan 2026 • ~3 min read • Python 3.1+
TL;DR

Counter is a dict subclass for tallying. Pass it any iterable, get back counts. Add or subtract counters, ask for .most_common(n), and skip the manual if x in d: d[x] += 1 dance forever.

Almost every codebase has a hand-rolled tally somewhere — a dict, a check for the key, an increment. Counter replaces that pattern with a one-liner, and adds a few tricks you'd never bother building yourself.

From iterable to counts in one call

pythonfrom collections import Counter

votes = ["yes", "no", "yes", "yes", "abstain", "no"]
tally = Counter(votes)
print(tally)
Counter({'yes': 3, 'no': 2, 'abstain': 1})

It works on any iterable — a list, a generator, the characters of a string, the words from a file. Missing keys return 0 instead of raising, so you can read tally["maybe"] safely.

The killer method: most_common

pythontext = "the quick brown fox jumps over the lazy dog the end"
freq = Counter(text.split())

print(freq.most_common(3))
[('the', 3), ('quick', 1), ('brown', 1)]

Returns a list of (item, count) tuples sorted by count descending. No argument means all items, sorted. This is the entire content of a hundred Stack Overflow answers about "top N words in a file."

Counters do arithmetic

Add two counters and the counts add. Subtract and they subtract, with negatives clamped to zero. & is min, | is max. This is wildly useful for diffs and rolling windows. 

pythonmonday  = Counter(errors=12, warnings=48, info=130)
tuesday = Counter(errors=8,  warnings=52, info=141)

print(monday + tuesday)
print(tuesday - monday)   # Day-over-day delta, no negatives
Counter({'info': 271, 'warnings': 100, 'errors': 20}) Counter({'info': 11, 'warnings': 4})

Updating in place

update() takes another iterable or mapping and adds — it doesn't overwrite like a regular dict. Easy to miss if you're coming from dict.update. 

pythontally = Counter()
for line in log_file:
    tally.update(line.split())     # adds to existing counts

Gotcha: Counter only counts hashable items. Lists and dicts won't work as keys. Convert to tuples first if you need to count sequences.
