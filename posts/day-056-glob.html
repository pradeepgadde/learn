glob vs pathlib.glob: pick your matcher

Two glob APIs, three patterns. ** for recursion, the case-sensitivity gotcha, and when to skip glob entirely for os.scandir.

25 Feb 2026 • ~3 min read • Python 3.5+
TL;DR

Use Path.rglob("*.py") for recursive matches — it's the cleanest API. The old-style glob.glob works too but returns strings; pathlib's version returns Path objects. For very large trees where you need attribute info too, os.scandir beats both.

Glob patterns are universal — everyone knows *.txt. Python has two ways to use them, and three patterns worth memorizing.

The patterns

  • * — any string of characters in a single path segment, except path separators.
  • ? — exactly one character.
  • [abc] — one character from the set.
  • ** — zero or more directories (recursive). Only when used as a full path segment.

pathlib.glob: the modern API

pythonfrom pathlib import Path

# All .py files in the current directory (non-recursive)
for p in Path(".").glob("*.py"):
    print(p)

# Recursive — every .py anywhere under the current dir
for p in Path(".").rglob("*.py"):
    print(p)
# rglob('*.py') is equivalent to glob('**/*.py')

Returns a generator of Path objects — lazy and memory-friendly.

The old API still works

pythonimport glob

glob.glob("*.py")               # list of strings
glob.iglob("**/*.py", recursive=True)   # lazy iterator

Mostly worth knowing because older codebases use it. New code: prefer pathlib.

The recursion gotcha: ** needs / separators

** only matches directory boundaries when it's a full path segment. **foo is just *foo. 

python# Recursive — works
Path(".").glob("**/*.py")

# NOT recursive — ** is just * here
Path(".").glob("**.py")

Case sensitivity bites cross-platform

*.py matches app.py on all platforms. *.PY matches app.PY on Linux but matches app.py on Windows and macOS (case-insensitive filesystems).

Python 3.12 added a case_sensitive= parameter to Path.glob for explicit control: 

python# Always case-sensitive, on every platform
Path(".").rglob("*.PY", case_sensitive=True)

# Always case-insensitive
Path(".").rglob("*.PY", case_sensitive=False)

Pre-3.12, the safe pattern is to either lowercase everything before comparing, or to glob with fnmatch against a known list.

What glob doesn't do: hidden files

By design, * doesn't match files starting with a dot. To get them too: 

python# Won't match .gitignore, .env, .DS_Store...
Path(".").glob("*")

# Either iterate explicitly...
[p for p in Path(".").iterdir()]

# ...or pattern them in
list(Path(".").glob(".*")) + list(Path(".").glob("*"))

For huge trees: os.scandir

Glob calls os.stat implicitly to decide what to recurse into. For trees with millions of files where you also need attributes (size, mtime, type), os.scandir gets all the metadata in one syscall per directory entry — dramatically faster. 

pythonimport os

def find_big_files(root, min_bytes=10 * 2**20):
    with os.scandir(root) as it:
        for entry in it:
            if entry.is_file(follow_symlinks=False):
                if entry.stat().st_size >= min_bytes:
                    yield entry.path
            elif entry.is_dir(follow_symlinks=False):
                yield from find_big_files(entry.path, min_bytes)

For most scripts, rglob is more than fast enough — reach for scandir only when you've measured a problem.

For pattern matching outside of file system contexts — glob-checking strings against patterns — use fnmatch from the stdlib. fnmatch.fnmatch("app.py", "*.py") returns True without touching the disk.

Find a script that combines os.walk with manual extension checks. Replace with Path(root).rglob( "*.ext"). Half the lines, same behavior, returns Path objects you can chain other pathlib calls onto.
