textwrap and difflib: text utilities you'll reach for

textwrap.dedent() cleans heredocs. textwrap.wrap() formats CLI output. difflib finds what changed between two texts — the engine behind git diff.

8 Apr 2026 • ~2 min read • Python 3.8+
TL;DR

textwrap.dedent() strips common leading whitespace from triple-quoted strings. textwrap.shorten() truncates with an ellipsis. difflib.unified_diff() produces a standard unified diff between two texts. difflib.get_close_matches() finds fuzzy matches in a list of strings.

textwrap.dedent: clean heredoc strings

pythonimport textwrap

# Without dedent, the indentation is part of the string:
query = """
    SELECT hostname, role
    FROM devices
    WHERE active = true
"""
# Has 4 leading spaces on every line

query = textwrap.dedent("""
    SELECT hostname, role
    FROM devices
    WHERE active = true
""").strip()
# No leading spaces, no leading/trailing blank lines

textwrap.wrap and shorten

pythonlong_desc = "This device handles traffic from the access layer and..."

# Wrap to 40 chars, return list of lines
lines = textwrap.wrap(long_desc, width=40)

# Wrap and join (fill is wrap + join)
print(textwrap.fill(long_desc, width=40))

# Truncate to N chars with ellipsis
textwrap.shorten(long_desc, width=30, placeholder="...")
# 'This device handles traffic...'

textwrap.indent: add a prefix to lines

pythoncode = "line1\nline2\nline3"
print(textwrap.indent(code, "    "))    # add 4 spaces to each line
# Conditional: only indent non-blank lines:
print(textwrap.indent(code, ">>> ", lambda line: line.strip()))

difflib.unified_diff: config change detection

pythonimport difflib

old_config = """interface gi0
 description uplink
 ip address 10.0.0.1 255.255.255.0
""".splitlines(keepends=True)

new_config = """interface gi0
 description core-uplink
 ip address 10.0.0.1 255.255.255.0
 shutdown
""".splitlines(keepends=True)

diff = difflib.unified_diff(
    old_config, new_config,
    fromfile="sw-01.before",
    tofile="sw-01.after",
    n=2,     # context lines
)
print("".join(diff))

Output is standard unified-diff format, the same format as git diff. Pipe it to rich.syntax.Syntax for coloured diffs in the terminal.

difflib.get_close_matches: fuzzy string lookup

pythonvendors = ["cisco", "arista", "juniper", "huawei"]

difflib.get_close_matches("csico", vendors)     # typo
# ['cisco']

# n= max matches, cutoff= similarity threshold (0-1)
difflib.get_close_matches("arst", vendors, n=2, cutoff=0.4)
# ['arista']

Useful in CLIs: when a user typos a subcommand, show "did you mean X?" using get_close_matches.

SequenceMatcher: similarity ratio

pythonmatcher = difflib.SequenceMatcher(None, "cisco", "csico")
matcher.ratio()    # 0.8 — 80% similar

# Detect suspiciously similar config stanzas
def similarity(a: str, b: str) -> float:
    return difflib.SequenceMatcher(None, a, b).ratio()

For network automation, unified_diff is invaluable for config-drift reports: "here is exactly what changed between the expected config and the device's running config." Pair with rich's syntax highlighting for a readable diff report in the terminal.

Take two versions of a network device config (or any two text files). Run difflib.unified_diff and print the result. Then add rich coloring: lines starting with + in green, lines starting with - in red. You now have a config-drift tool in twenty lines.
