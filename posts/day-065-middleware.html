HTTP middleware: cross-cutting concerns the right way

Middleware wraps every request. Use it for request IDs, timing, CORS, and trusted hosts — but not auth (dependencies handle that better).

06 Mar 2026 • ~3 min read • Python 3.8+
TL;DR

Add middleware with @app.middleware("http") on a function that calls response = await call_next(request). Use built-in middleware for CORS, gzip, trusted hosts. For per-request auth, prefer FastAPI dependencies — they integrate with the OpenAPI schema and run after path matching.

Middleware in ASGI runs once per request, wrapping the entire downstream stack. It's the right tool for things that should apply uniformly — request IDs, timing, security headers — and the wrong tool for things that should differ per endpoint.

The shape of FastAPI middleware

pythonfrom fastapi import FastAPI, Request
from time import perf_counter
import uuid

app = FastAPI()

@app.middleware("http")
async def add_request_id(request: Request, call_next):
    request.state.request_id = request.headers.get(
        "X-Request-ID", uuid.uuid4().hex
    )
    t0       = perf_counter()
    response = await call_next(request)
    response.headers["X-Request-ID"]   = request.state.request_id
    response.headers["X-Process-Time"] = f"{(perf_counter() - t0) * 1000:.1f}ms"
    return response

Three things to note. The function is async def; request.state is the per-request scratch space (use it instead of globals); call_next is what triggers downstream processing.

Built-in middleware worth turning on

pythonfrom fastapi.middleware.cors            import CORSMiddleware
from fastapi.middleware.gzip            import GZipMiddleware
from starlette.middleware.trustedhost    import TrustedHostMiddleware
from starlette.middleware.httpsredirect  import HTTPSRedirectMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://app.example.com"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

app.add_middleware(GZipMiddleware, minimum_size=1000)

app.add_middleware(
    TrustedHostMiddleware,
    allowed_hosts=["api.example.com", "*.internal"],
)

TrustedHostMiddleware rejects requests with unexpected Host headers — a cheap defense against host-header injection.

Order matters

Middleware runs in the reverse of registration order on the way in, then in registration order on the way out. Add the outermost behavior first. 

python# Outermost (runs first incoming, last outgoing)
app.add_middleware(CORSMiddleware, ...)

# Then trusted host check
app.add_middleware(TrustedHostMiddleware, ...)

# Innermost — closest to the path function
@app.middleware("http")
async def add_request_id(request, call_next):
    ...

Why dependencies, not middleware, for auth

Middleware runs before path matching, so it applies to every request — including OPTIONS preflights, health checks, the docs page. Per-endpoint auth via Depends(require_user) shows up in the OpenAPI schema, runs only on relevant routes, and is testable per-endpoint. 

python# Use dependencies for auth
def require_user(authorization: str = Header(...)):
    ...validate, return user...

@app.get("/me")
def me(user = Depends(require_user)):
    return user

Logging context: bind the request ID

Pair the request-ID middleware with structlog (Day 034) so every log line in the request carries the ID automatically. 

pythonfrom structlog.contextvars import bind_contextvars, clear_contextvars

@app.middleware("http")
async def log_context(request, call_next):
    clear_contextvars()
    bind_contextvars(
        request_id=request.state.request_id,
        path=request.url.path,
        method=request.method,
    )
    return await call_next(request)

Now every log.info(...) in any downstream code automatically includes the request ID without threading it through.

Don't read the body in middleware unless you also re-set it. Reading drains the stream and the path function gets nothing. If you need the body for logging or signing, buffer and re-stream — and pay the memory cost.

Add the request-ID + timing middleware above to a FastAPI service. Tail the logs while you hit it. Every line should now include a request ID; you should be able to filter for one specific request through the whole stack with one grep.
