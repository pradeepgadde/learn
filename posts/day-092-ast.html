ast: parse and inspect Python source

Turn Python source into a tree you can walk. Lint rules, code generators, safe eval replacements — all start here.

2 Apr 2026 • ~3 min read • Python 3.8+
TL;DR

ast.parse(source) returns a tree of node objects. Walk it with ast.walk() or subclass ast.NodeVisitor for structured traversal. Use ast.literal_eval() as a safe replacement for eval() on simple data literals.

Everything Python tooling does — ruff, mypy, black, coverage — starts by parsing source into an AST. The ast module exposes that same tree to you.

Parse and inspect

pythonimport ast

source = """
def greet(name: str) -> str:
    return f"hello, {name}"
"""

tree = ast.parse(source)
print(ast.dump(tree, indent=2))

ast.dump pretty-prints the whole tree. Each node has a type and fields; for example a FunctionDef node has name, args, body, decorator_list, and returns.

Walk the tree: find all function names

pythonimport ast, pathlib

source = pathlib.Path("myapp/core.py").read_text()
tree   = ast.parse(source)

functions = [
    node.name
    for node in ast.walk(tree)
    if isinstance(node, (ast.FunctionDef, ast.AsyncFunctionDef))
]
print(functions)

NodeVisitor: structured traversal

For anything more complex than a flat scan, subclass NodeVisitor. Define visit_NodeType methods for the node types you care about. 

pythonclass ImportFinder(ast.NodeVisitor):
    def __init__(self):
        self.imports = []

    def visit_Import(self, node):
        for alias in node.names:
            self.imports.append(alias.name)
        self.generic_visit(node)       # recurse into children

    def visit_ImportFrom(self, node):
        self.imports.append(node.module)
        self.generic_visit(node)

finder = ImportFinder()
finder.visit(tree)
print(finder.imports)

ast.literal_eval: safe eval for data

Never use eval() on untrusted input. ast.literal_eval parses only Python literals — strings, numbers, dicts, lists, tuples, booleans, None. Anything else raises ValueError. 

pythonast.literal_eval('{"host": "sw-01", "port": 22}')
# {'host': 'sw-01', 'port': 22}

# Safe — raises ValueError, doesn't execute:
ast.literal_eval("__import__('os').system('rm -rf /')")
# ValueError: malformed node or string

NodeTransformer: rewrite the tree

NodeTransformer is like NodeVisitor but can return a modified node to replace the original. Use it to build code generators or AST macros. 

pythonclass DebugPrinter(ast.NodeTransformer):
    """Wrap every return statement in a print()."""
    def visit_Return(self, node):
        print_call = ast.Expr(value=ast.Call(
            func=ast.Name(id="print", ctx=ast.Load()),
            args=[node.value], keywords=[],
        ))
        ast.fix_missing_locations(print_call)
        return [print_call, node]   # insert before return

Write a simple lint rule

Custom lint checks are just NodeVisitors. For example: flag any print() calls left in a module. 

pythonclass NoPrintLint(ast.NodeVisitor):
    def visit_Call(self, node):
        if isinstance(node.func, ast.Name) and node.func.id == "print":
            print(f"Line {node.lineno}: print() found")
        self.generic_visit(node)

NoPrintLint().visit(ast.parse(pathlib.Path("app.py").read_text()))

If you're writing a custom ruff plugin or flake8 extension, you're writing an ast.NodeVisitor. Understanding the AST is the prerequisite for any serious Python tooling work.

Run ast.dump(ast.parse(open("your_script.py").read()), indent=2) on one of your own files. Read the output. Find a FunctionDef, look at its children. Once the tree structure makes sense, write a visitor that counts function definitions per file across a directory.
