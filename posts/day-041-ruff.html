ruff: the linter that ate the ecosystem

Replaces flake8, isort, pyupgrade, autoflake, and most of pylint — running 10–100× faster. Plus a formatter that's a black drop-in.

10 Feb 2026 • ~3 min read • Any Python
TL;DR

ruff check lints your code, ruff check --fix auto-fixes what it can, ruff format formats. Configure with a small [tool.ruff] block in pyproject.toml. Wire it into pre-commit and CI for free safety.

For ten years the Python tooling story was: install flake8, isort, pyupgrade, black, autoflake, pylint — configure each, hope they didn't fight each other, watch CI take 90 seconds to lint a small repo. ruff (Rust-based, from the same team as uv) replaced all of them with one binary that runs in 0.5 seconds.

Install and run

bashpip install ruff
# Or, with uv (Day 019)
uv add --dev ruff

# Lint your code
ruff check .

# Lint and auto-fix what's safe
ruff check --fix .

# Format (black-compatible)
ruff format .

First run on a moderately-sized codebase usually takes a couple of seconds. Subsequent runs are near-instant because ruff caches.

Configuration in pyproject.toml

toml# pyproject.toml
[tool.ruff]
line-length = 100
target-version = "py312"

[tool.ruff.lint]
# Pick rule sets by short prefix:
#   E,W = pycodestyle
#   F   = pyflakes
#   I   = isort
#   UP  = pyupgrade
#   B   = bugbear (real bug catchers)
#   SIM = flake8-simplify
#   N   = pep8-naming
select = ["E", "W", "F", "I", "UP", "B", "SIM", "N"]
ignore = ["E501"]   # line too long — formatter handles it

[tool.ruff.lint.per-file-ignores]
"tests/*" = ["S101"]   # allow assert in tests

The configuration is genuinely small. Compare to maintaining .flake8, pyproject.toml for isort, setup.cfg for pyupgrade, etc.

The auto-fix is what changes your workflow

ruff check --fix handles unused imports, wrong import order, deprecated syntax (List[int] → list[int]), and dozens of other categories without you doing anything. Run it once on an old codebase and watch hundreds of small fixes apply cleanly. 

bashruff check --fix --unsafe-fixes .
# --unsafe-fixes opts in to fixes that are mostly-safe but
# could change behavior. Always review the diff afterwards.

The formatter

ruff format is a near-perfect re-implementation of black in Rust. Same opinionated style, same output (mostly). If you're already on black, the migration is "delete black, install ruff." If you're on neither, just start with ruff. 

bashruff format .          # reformat in place
ruff format --check .  # CI mode: exit non-zero if anything would change

pre-commit integration

The right time to lint is before the commit lands. The pre-commit framework runs ruff (and other checks) on staged files automatically. 

yaml# .pre-commit-config.yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.6.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
bashpip install pre-commit
pre-commit install
# Now every git commit runs ruff first

The rule prefixes worth knowing

  • F — pyflakes: real correctness checks (unused imports, undefined names). Always on.
  • E, W — pycodestyle: style. Mostly handled by the formatter, but some sub-rules still matter.
  • I — isort: import sorting. Auto-fixable.
  • UP — pyupgrade: rewrites old syntax to modern (%-format → f-string, List[int] → list[int]). Auto-fixable.
  • B — bugbear: real bug catchers (mutable default args, raising bare exceptions). Worth its weight.
  • SIM — simplify: suggests cleaner equivalents (if x: return True; else: return False → return bool(x)).

ruff covers about 800 rules at the time of writing — including the entire pylint convention/refactor/warning rule sets. For projects that historically used pylint (which was always slower and configured differently), ruff is a near-complete replacement now.

On any project: uv add --dev ruff, drop the config block above into pyproject.toml, run ruff check --fix . && ruff format .. Review the diff. Commit. You've just paid down weeks of accumulated style debt in two minutes.
