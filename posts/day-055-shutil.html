shutil: the file-operations toolbox

copy, move, rmtree, archive making, disk usage, terminal size — the swiss-army knife for file-system operations.

24 Feb 2026 • ~3 min read • Python 3.0+
TL;DR

shutil.copy copies a single file, copytree recurses, rmtree deletes a tree, move renames or copies-then-deletes as needed. Plus archive helpers (make_archive) and a few system introspection tools (which, disk_usage, get_terminal_size).

pathlib (Day 002) handles single-file operations beautifully but stops short of recursive tree work. That's where shutil comes in — the operations everyone needs but nobody wants to write by hand.

Copy and move

pythonimport shutil

# Copy one file (preserves metadata: timestamps, permissions)
shutil.copy2("src.cfg", "backup/src.cfg")

# Recursively copy a tree
shutil.copytree("templates/", "build/templates/")

# Move/rename — works across filesystems (copies if rename can't)
shutil.move("old/", "archive/2026/")

copy, copy2, copyfile, copytree — the differences:

  • copyfile — copies bytes only, no metadata.
  • copy — copies bytes plus permissions.
  • copy2 — copies everything copy does plus timestamps. Closest to cp -p. Use this by default.
  • copytree — recursive directory copy. Uses copy2 per file by default.

rmtree: the recursive delete

python# Delete a tree, ignoring "doesn't exist" errors
shutil.rmtree("build/", ignore_errors=True)

The dangerous one. There is no undo. Type the path twice in your head before you type it once in your editor. 

python# For removing read-only files on Windows you may need an onerror handler:
def on_remove_error(func, path, exc):
    os.chmod(path, stat.S_IWRITE)
    func(path)

shutil.rmtree("protected/", onerror=on_remove_error)

Archives: make and unpack

Need a tarball or zip? Don't shell out, don't import tarfile for the common cases. 

python# Create build/release.tar.gz from the build/ directory
shutil.make_archive("release", "gztar", root_dir="build")
# Output: release.tar.gz in cwd

# Other formats: 'zip', 'tar', 'bztar', 'xztar'

# Unpack any supported format — auto-detects from extension
shutil.unpack_archive("release.tar.gz", "extracted/")

Finding executables: which

Like the shell command. Useful when your script needs to verify a tool is available before invoking subprocess.run. 

pythonif shutil.which("git") is None:
    sys.exit("git is required but not installed")

# Returns the full path: '/usr/bin/git' or None

Disk usage and terminal size

python# Disk space at any path — total, used, free in bytes
total, used, free = shutil.disk_usage("/")
print(f"{free / 2**30:.1f} GB free")

# Terminal width and height — for fitting CLI output
cols, rows = shutil.get_terminal_size()

Both honor environment variables (COLUMNS, LINES) and degrade gracefully when running without a TTY (CI, pipes).

Streaming copy with progress

For very large files where you want a progress indicator, use copyfileobj with chunked reads. 

pythondef copy_with_progress(src, dst, callback):
    total = os.stat(src).st_size
    copied = 0
    with open(src, "rb") as s, open(dst, "wb") as d:
        while chunk := s.read(1024 * 1024):
            d.write(chunk)
            copied += len(chunk)
            callback(copied, total)

Pair with rich.progress from Day 031 for a slick progress bar over big copies.

Permissions caveat: on macOS, shutil.copytree doesn't preserve extended attributes by default. For system-restore use cases (where ACLs and xattrs matter), use cp -a via subprocess, or the copy_function parameter to plug in your own.

Replace any os.system("cp -r ...") or subprocess.run(["cp", "-r", ...]) calls in your scripts with shutil.copytree. Cross-platform for free, no shell quoting bugs, no PATH surprises.
