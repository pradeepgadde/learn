dict.get and the missing-key dance

Three tools, three different jobs. Knowing which to reach for is what separates clean code from a forest of if key in d.

04 Jan 2026 • ~3 min read • Python 3.0+
TL;DR

d.get(k, default) reads safely. d.setdefault(k, default) reads-or-inserts. defaultdict(list) auto-creates the default the first time you touch a key. Pick whichever matches your intent.

Missing keys raise KeyError, which is the right default — loud failures beat silent bugs. But sometimes you genuinely have an absent-means-default situation, and the idiomatic Python for it is shorter than what most people write.

Just reading: get

get never raises. Pass a default, get it back when the key isn't there. Without a default, you get None. 

pythonconfig = {"host": "localhost", "port": 8080}

timeout = config.get("timeout", 30)
debug   = config.get("debug", False)

# Compare the verbose version that's everywhere:
timeout = config["timeout"] if "timeout" in config else 30

Two important things get does not do: it doesn't insert anything into the dict, and it doesn't help if the value is present-but-None. For that, you want a real or or an explicit is None check.

Reading or inserting: setdefault

setdefault is the operation people don't realize they need. It returns the existing value if the key is there; otherwise it inserts the default and returns it. One call, one lookup, no race. 

pythongroups = {}
for user, team in memberships:
    groups.setdefault(team, []).append(user)

# groups is now {'platform': ['ada','linus'], 'data': ['guido'], ...}

That's the entire group-by pattern. The list is created on first sight of each team, and appended to on every visit afterward.

The lazier option: defaultdict

If almost every read also inserts, defaultdict skips the setdefault step. Pass it a factory function (the type itself works as a factory: list, set, int) and missing keys are auto-populated. 

pythonfrom collections import defaultdict

groups = defaultdict(list)
for user, team in memberships:
    groups[team].append(user)

# int as factory gives you a free counter:
tally = defaultdict(int)
for word in text.split():
    tally[word] += 1

For pure tallying, prefer Counter from Day 003 — it's a defaultdict(int) with superpowers. defaultdict shines when the factory is anything other than int.

Choosing between them

The mental model that has served me well:

  • Reading only? Use get.
  • One-off read-or-insert? Use setdefault.
  • Many read-or-inserts in a loop? Use defaultdict.
  • Counting? Use Counter.

Search your codebase for if .* in in dict contexts. Most of those can become get, setdefault, or a defaultdict. Pick one and refactor it.
