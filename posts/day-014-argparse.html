A real CLI in 12 lines of argparse

You don't need Click or Typer for a useful CLI. The standard library has been good enough since 2011.

14 Jan 2026 • ~3 min read • Python 3.2+
TL;DR

argparse in 12 lines gives you positional and optional arguments, type coercion, validated choices, default values, and an auto-generated --help. Reach for a third-party CLI library only when you need subcommands or Rich-style output.

Every networking and DevOps tool starts as a script that takes a hostname. Six months later it has six flags and no --help. argparse prevents that future, and the minimal version is shorter than most people think.

The 12-line template

pythonimport argparse

def main():
    p = argparse.ArgumentParser(
        description="Probe a router and print interface stats.",
    )
    p.add_argument("host", help="hostname or IP of the router")
    p.add_argument("--port",    type=int, default=22)
    p.add_argument("--timeout", type=float, default=5.0)
    p.add_argument("--format", choices=["json", "table"], default="table")
    p.add_argument("--verbose", "-v", action="store_true")
    args = p.parse_args()
    probe(args.host, args.port, args.timeout, args.format, args.verbose)

if __name__ == "__main__":
    main()

That's it. Now run it with no args and watch:

$ python probe.py usage: probe.py [-h] [--port PORT] [--timeout TIMEOUT] [--format {json,table}] [--verbose] host probe.py: error: the following arguments are required: host $ python probe.py --help usage: probe.py [-h] [--port PORT] [--timeout TIMEOUT] ... Probe a router and print interface stats. positional arguments: host hostname or IP of the router options: -h, --help show this help message and exit --port PORT --timeout TIMEOUT --format {json,table} --verbose, -v

The four argument flavors you actually need

  • Positional: p.add_argument("host")
  • Optional with value: p.add_argument("--port", type=int, default=22)
  • Boolean flag: p.add_argument("--verbose", action="store_true")
  • Repeatable list: p.add_argument("--tag", action="append", default=[])

With those four patterns you can describe almost any CLI in the wild.

Free validation: type and choices

type is any callable that takes a string and returns the parsed value. int, float, and pathlib.Path all qualify, and so does any function you write. 

pythonfrom pathlib import Path

def existing_file(s):
    p = Path(s)
    if not p.is_file():
        raise argparse.ArgumentTypeError(f"no such file: {s}")
    return p

parser.add_argument("--config", type=existing_file, required=True)

Subcommands, when you grow into them

When the script has multiple distinct verbs — git commit, git push, kubectl get, kubectl apply — subparsers keep them organized. 

pythonp   = argparse.ArgumentParser()
sub = p.add_subparsers(dest="cmd", required=True)

show = sub.add_parser("show", help="display config")
show.add_argument("section")

apply = sub.add_parser("apply", help="push config")
apply.add_argument("--dry-run", action="store_true")

args = p.parse_args()

For a single-file script, argparse is fine. If you find yourself wanting subcommand groups, rich help formatting, and color, that's when typer or click earn their dependency. But many "we need a CLI library" decisions are premature.

Pick a one-off script you have that uses sys.argv directly. Convert it to argparse using the 12-line template. Add --help documentation to each argument as you go — you'll be glad in three months.
