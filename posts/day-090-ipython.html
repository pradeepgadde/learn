IPython and the REPL toolkit: March finale

The REPL is where Python thinking happens. IPython, rich.inspect(), %timeit, %debug, and the habits that make exploration fast and results reproducible.

31 Mar 2026 • ~3 min read • Python 3.8+
TL;DR

IPython gives you tab completion, magic commands (%timeit, %debug, %run), object introspection with ?, and a history that outlasts the session. Pair with rich.inspect() for a pretty-printed object explorer. Understand the REPL as a first-class thinking tool, not a fallback for when you don't want to write a test.

Python's default REPL is fine; IPython is excellent. After ninety days of stdlib, third-party libraries, CI, Docker, and packaging, it's worth closing the month on the tool you'll actually use to explore all of it: the interactive shell.

Getting started

bashuv tool install ipython
ipython

Or inside a project: 

bashuv add --dev ipython
uv run ipython

? and ?? — instant docs

pythonIn [1]: import pathlib

In [2]: pathlib.Path.glob?           # shows docstring
In [3]: pathlib.Path.glob??          # shows source code

The ?/?? shorthand is the fastest way to check a signature or read an implementation without leaving the REPL. Works on functions, classes, modules, and instances.

Tab completion and autoreload

Tab completes attribute names, path strings (inside Path("")), and dict keys. For iterative script development, the autoreload extension reloads changed modules automatically: 

pythonIn [1]: %load_ext autoreload
In [2]: %autoreload 2         # reload all modules before execution
In [3]: from myapp import core

# Edit myapp/core.py in your editor, then call core.my_func()
# without restarting IPython — changes appear immediately

%timeit — quick benchmarks

pythonIn [4]: %timeit sorted(range(1000))
# 18.5 μs ± 312 ns per loop (mean ± std. dev. of 7 runs, 100,000 loops each)

# Multi-line variant:
In [5]: %%timeit
   ...: data = list(range(1000))
   ...: sorted(data)

Runs the expression in a tight loop, auto-scales iterations, reports mean ± standard deviation. The fastest way to compare two implementations without writing a benchmark script.

%debug — post-mortem debugging

pythonIn [6]: parse_cidr("not-an-ip")
# ValueError: 'not-an-ip' does not appear to be an IPv4 or IPv6 network

In [7]: %debug
# Opens the debugger AT the frame where the exception occurred
# Full pdb — u/d to move frames, p to print, l to list source

Post-mortem debugging drops you inside the traceback to inspect locals. Usually faster than adding print() statements and rerunning.

%run — run a script in the REPL namespace

pythonIn [8]: %run my_script.py
# All top-level names from the script are now in the REPL
# Great for "run a data pipeline, then explore the results"

rich.inspect() — pretty object explorer

When ? isn't pretty enough, rich.inspect gives you a coloured table of all attributes and methods: 

pythonfrom rich import inspect
import httpx

r = httpx.get("https://httpbin.org/get")
inspect(r)                             # attributes + methods
inspect(r, methods=True)               # include all methods
inspect(r, private=True)              # include _ prefixed attrs

Much friendlier than dir(r) for exploring unfamiliar objects from a new library.

Configuring IPython

One-time setup: generate a profile, then edit ~/.ipython/profile_default/ipython_config.py: 

python# Auto-load autoreload and rich on startup:
c.InteractiveShellApp.extensions = ['autoreload']
c.InteractiveShellApp.exec_lines = [
    '%autoreload 2',
    'from rich import inspect, print',
]

Now every new IPython session starts with autoreload and rich.print ready.

A quick workflow note

The REPL is for exploration; tests are for codifying what you found. A good pattern: explore an idea in IPython, verify it works, then write it as a test so it stays working. The REPL is the scratch pad; the test suite is where conclusions live.

Three months in. Ninety days, ninety ideas — from f-strings to property tests to type stubs. The breadth was deliberate: this is the vocabulary that separates Python users from Python practitioners. April continues.

Open IPython (or install it today if you haven't). Run %timeit on two ways you'd normally do the same thing — a list comprehension vs a loop, a set lookup vs a linear scan. Let the numbers inform the decision rather than intuition. That's the culture the REPL encourages.
