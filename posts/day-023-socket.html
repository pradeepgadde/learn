socket: the bare metal of TCP

The foundation under every HTTP library. A TCP port check is six lines — and the day a library fails you, knowing sockets is what gets you out of trouble.

23 Jan 2026 • ~3 min read • Python 3.0+
TL;DR

Use socket.create_connection((host, port), timeout) for any TCP probe — it handles DNS, IPv4/IPv6 fallback, and timeouts in one call. For DNS lookups, socket.getaddrinfo beats gethostbyname because it returns IPv6 results too.

You'll rarely write socket code in a web app, but for network probes, port scanners, custom protocols, and service health checks, the socket module is exactly the right level of abstraction.

The TCP port check, the right way

pythonimport socket

def tcp_open(host, port, timeout=3.0):
    try:
        with socket.create_connection((host, port), timeout):
            return True
    except (socket.timeout, OSError):
        return False

tcp_open("github.com", 443)   # True
tcp_open("github.com", 9999)  # False

create_connection resolves DNS, tries each returned address (IPv4 and IPv6), respects the timeout, and cleans up on failure. The context manager closes the socket even on exception. Six lines, no leaked file descriptors.

The DNS call you should be making

Most code uses socket.gethostbyname. It returns a single IPv4 address and ignores IPv6 entirely. getaddrinfo returns everything DNS knows. 

pythonimport socket

for family, _, _, _, sockaddr in socket.getaddrinfo(
    "google.com", 443, type=socket.SOCK_STREAM
):
    label = "IPv6" if family == socket.AF_INET6 else "IPv4"
    print(label, sockaddr[0])
IPv6 2607:f8b0:4005:80b::200e IPv4 142.250.190.78

Filtering by type=SOCK_STREAM dedupes UDP results so you don't see each address twice.

A minimal "send a line, read a line" client

For a custom TCP protocol — a service exposing a line-oriented interface, an SMTP probe, an HTTP GET sent by hand — the socket file-object trick keeps it readable. 

pythonwith socket.create_connection(("smtp.gmail.com", 25), timeout=5) as s:
    f = s.makefile("rwb", buffering=0)
    print(f.readline().decode().strip())     # server banner
    f.write(b"EHLO probe.local\r\n")
    while line := f.readline():
        print(line.decode().strip())
        if not line.startswith(b"250-"):
            break

Get the local IP a route would actually use

A favourite trick: ask the kernel which interface would be chosen for a given destination, without sending any traffic. UDP connect is purely local — it just consults the routing table. 

pythondef local_ip_for(dest="8.8.8.8"):
    with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as s:
        s.connect((dest, 80))
        return s.getsockname()[0]

print(local_ip_for())   # your egress IP for the default route

Way more reliable than socket.gethostbyname(socket.gethostname()), which routinely returns 127.0.1.1 on Debian-family hosts.

Always set a timeout. A socket without one waits as long as the OS is willing to (often minutes). Either pass timeout= to create_connection or call s.settimeout(N) before any read.
