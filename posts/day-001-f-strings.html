The hidden grammar of f-strings

You've used f"{name}" a thousand times. The mini-language inside the braces is where the real power lives.

01 Jan 2026 • ~3 min read • Python 3.8+
TL;DR

Everything after the colon in f"{value:spec}" is a tiny formatting language. Learn five symbols — =, >, <, ^, , — and your debug prints, log lines, and CLI tables get dramatically nicer.

f-strings are the most-used feature in modern Python, and the most under-used. Most code stops at f"Hello {name}". But the colon opens a door — behind it is the same format spec that str.format() has had since 2008, now with the syntax it always deserved.

The = debug shortcut

Since Python 3.8, putting = after a name prints both the name and the value. This is the single biggest quality-of-life upgrade for print-debugging in years. 

pythonuser_id = 42
scores = [88, 91, 76]

print(f"{user_id=}")
print(f"{scores=}")
print(f"{sum(scores) / len(scores)=:.2f}")
user_id=42 scores=[88, 91, 76] sum(scores) / len(scores)=85.00

Notice the third one: the = works on whole expressions, and you can still chain a format spec after it. The next time you reach for a sad print("scores:", scores), reach for this instead.

Padding and alignment

Three single characters control alignment inside a fixed width: > right, < left, ^ center. Put the width after them. Put a fill character before them. 

pythonfor name, role in [("Ada", "founder"),
                  ("Linus", "maintainer"),
                  ("Guido", "BDFL")]:
    print(f"{name:<10}{role:>12}")
Ada founder Linus maintainer Guido BDFL

Want a dotted leader, like a table of contents?

pythonprint(f"{'Chapter 1':.<30}{12}")
print(f"{'Chapter 2':.<30}{47}")
Chapter 1.....................12 Chapter 2.....................47

Numbers that read like a human wrote them

Three specs cover ninety percent of what you need: , for thousands separators, .Nf for fixed decimals, .N% for percentages. 

pythonrevenue   = 1_482_330.7
conversion = 0.04217

print(f"Revenue:    ${revenue:,.2f}")
print(f"Conversion: {conversion:.1%}")
print(f"Hex of 255: {255:#06x}")
Revenue: $1,482,330.70 Conversion: 4.2% Hex of 255: 0x00ff

Dates, for free

f-strings forward the format spec to the object's __format__ method. datetime uses it to accept strftime codes directly — no separate call. 

pythonfrom datetime import datetime

now = datetime.now()
print(f"{now:%Y-%m-%d}")        # 2026-05-02
print(f"{now:%A, %B %d}")       # Saturday, May 02
print(f"Logged at {now:%H:%M:%S}")

One pro move: nested format specs

The width or precision can itself be an expression in braces. Useful when you don't know the column size until runtime. 

pythonwidth = max(len(w) for w in words) + 2
for w in words:
    print(f"{w:<{width}}{len(w)}")

The full grammar lives in the Format Specification Mini-Language docs. Bookmark that page; it pays off.

Take a script in your codebase that has at least one print("debug:", x, y, z)-style line and rewrite it using f"{x=}, {y=}, {z=}".

Then find a place where you build a string with str(round(n, 2)) and collapse it down to f"{n:.2f}". You'll usually delete a line or two along the way.
