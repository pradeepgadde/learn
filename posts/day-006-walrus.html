The walrus that earns its keep

:= is divisive. Used in the right two or three places, it's a code-shrinking, intent-clarifying gem.

06 Jan 2026 • ~3 min read • Python 3.8+
TL;DR

The walrus operator := assigns and returns a value in the same expression. It shines in three spots: while loops that compute-then-test, comprehensions that need to reuse an expensive value, and any conditional that wants to inspect what it just computed.

Introduced in PEP 572 and immediately controversial, the walrus operator deserves a quieter reputation. It's not for sprinkling everywhere — but it kills three specific code smells dead.

The compute-then-test loop

The classic case: read a chunk, check if it's empty, process it. Without walrus you read in two places to avoid an infinite loop. 

python# Before
chunk = f.read(8192)
while chunk:
    process(chunk)
    chunk = f.read(8192)

# After
while chunk := f.read(8192):
    process(chunk)

One read site, no duplication, no risk of the prime and the loop drifting out of sync when someone edits one and forgets the other.

Reuse inside a comprehension

When a list comprehension filters on an expensive expression and also needs the result, you usually call it twice: 

python# Calls expensive_check twice per item
results = [expensive_check(x) for x in items
           if expensive_check(x) is not None]

# Once, with walrus
results = [y for x in items
           if (y := expensive_check(x)) is not None]

Inspect what you just matched

Regex matches are a perfect fit. You want the match object if it matched, and to skip otherwise. 

pythonimport re

for line in log:
    if m := re.match(r"ERROR (\d+): (.+)", line):
        code, msg = m.groups()
        handle(code, msg)

Compare to the assignment-then-test version, which leaks m into the surrounding scope and reads less directly.

When NOT to use it

The walrus loses its value when there's no duplication to eliminate. These are not improvements: 

python# Don't — just use a normal assignment
(x := 42)

# Don't — clarity loss for no gain
if (total := sum(items)):
    print(total)
# Two lines is fine; the walrus is overhead here.

Rule of thumb: reach for := only when you'd otherwise repeat an expression or maintain a "prime the loop" pattern. Anywhere else, a plain assignment on its own line is more readable.

Find a while True: loop in your code that calls the same function on the first line and at the bottom of the loop. Collapse it with :=.
