pytest fixtures: scope, factories, and yield

Move past simple return fixtures. yield for setup/teardown, factories for parametric setup, scope for performance, conftest for sharing.

18 Mar 2026 • ~3 min read • Python 3.8+
TL;DR

Use yield in fixtures to do cleanup after the test (finally not needed — pytest handles it). Use scope="session" or "module" to share expensive setup. Use factory fixtures (a fixture that returns a function) when each test needs a slightly different setup. Put shared fixtures in conftest.py at the right level of your tree.

Day 018 covered the basics. The five patterns below are what take a test suite from "fine" to "I'm not afraid to write more tests".

yield for setup/teardown

pythonimport pytest
from myapp import create_app

@pytest.fixture
def tmp_db(tmp_path):
    db_path = tmp_path / "test.db"
    conn    = connect(db_path)
    migrate(conn)
    yield conn                  # test runs here
    conn.close()                  # cleanup, even on test failure

Pytest catches exceptions in the test, runs the teardown, then re-raises. Cleaner than setUp/tearDown; equivalent in safety to try/finally.

Scope: don't pay setup cost per test

Spinning up a database, loading a model file, starting a Docker container — you don't want to do that 200 times for 200 tests. Set the scope. 

python@pytest.fixture(scope="session")
def db_engine():
    # created once for the whole test session
    engine = create_engine("postgresql://...")
    Base.metadata.create_all(engine)
    yield engine
    engine.dispose()

@pytest.fixture      # default: function scope — one per test
def db_session(db_engine):
    # Wrap each test in a transaction we roll back
    conn = db_engine.connect()
    trans = conn.begin()
    session = Session(bind=conn)
    yield session
    session.close()
    trans.rollback()
    conn.close()

Engine spun up once; each test gets a fresh session inside a transaction that gets rolled back at the end. Tests are isolated and fast.

Scope options: function (default), class, module, package, session. Higher scope = less isolation but faster.

Factory fixtures: parametric setup

Sometimes the test needs to create the thing with different inputs. A regular fixture would force you to define one fixture per configuration. Factory fixtures fix this. 

python@pytest.fixture
def make_device(db_session):
    created = []
    def _factory(**overrides):
        defaults = {"hostname": "sw-test", "role": "access"}
        d        = Device(**{**defaults, **overrides})
        db_session.add(d)
        db_session.flush()
        created.append(d)
        return d
    yield _factory
    # cleanup uses the closed-over `created` list

# In a test:
def test_role_filter(make_device, db_session):
    make_device(role="access")
    make_device(role="core")
    assert count_by_role(db_session, "access") == 1

Each test gets full control over what to create without writing its own boilerplate.

conftest.py: shared fixtures, scoped to subtree

Fixtures in tests/conftest.py are available to every test below it, no import needed. Fixtures in tests/api/conftest.py are available only to tests under tests/api/. The hierarchical layout naturally scopes shared setup. 

texttests/
├── conftest.py              # db_engine, settings — shared everywhere
├── unit/
│   ├── conftest.py          # mock_clock, fake_clients — unit-only
│   └── test_*.py
└── integration/
    ├── conftest.py          # real_clients, docker_compose — integration-only
    └── test_*.py

autouse: when you don't want to opt in

python@pytest.fixture(autouse=True)
def silence_logs(caplog):
    caplog.set_level(logging.CRITICAL)

autouse=True applies the fixture to every test in scope without listing it as a parameter. Use sparingly — it makes test setup invisible at the test level.

Built-in fixtures worth knowing

  • tmp_path — fresh Path for the test, auto-cleaned up.
  • monkeypatch — safe attribute/env-var mocking, undone after the test.
  • caplog — capture log records, assert what was logged.
  • capsys / capfd — capture stdout/stderr at Python or FD level.
  • request — fixture metadata; lets you inspect the test that's using you.

For environment-variable mocking specifically: monkeypatch.setenv("KEY", "value"). The env is reset after the test, so test ordering doesn't leak. Beats os.environ["KEY"] = "..." + forgetting to un-set.

Find a test file with repeated setup at the top of every test. Hoist it into a fixture in conftest.py. Add a factory variant for the cases that need slight differences. The test bodies should drop down to just the assertion + minimum setup — the part that's actually unique.
