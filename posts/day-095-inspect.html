inspect: introspect functions, classes, and frames

Read live Python objects: signatures, source code, call stacks, class hierarchies. The foundation of most Python framework magic.

5 Apr 2026 • ~3 min read • Python 3.8+
TL;DR

inspect.signature(fn) gives you parameter names and defaults. inspect .getsource(obj) returns the source. inspect.stack() walks the call stack. FastAPI uses inspect.signature to derive dependency injection from function signatures — this is how that magic works.

Signatures: parameter names and defaults

pythonimport inspect

def connect(host: str, port: int = 22, timeout: float = 30.0) -> None: ...

sig = inspect.signature(connect)

for name, param in sig.parameters.items():
    print(name, param.annotation, param.default)
# host  <class 'str'>   <class 'inspect._empty'>
# port  <class 'int'>   22
# timeout <class 'float'> 30.0

# Bind arguments to parameters (validates arity):
bound = sig.bind("10.0.0.1", timeout=10.0)
bound.apply_defaults()
bound.arguments   # {'host': '10.0.0.1', 'port': 22, 'timeout': 10.0}

This is exactly what FastAPI does: it reads your endpoint function's signature, notices the parameters, and wires up the query params, path variables, and Depends() accordingly.

Type predicates: is this a class? async? coroutine?

pythoninspect.isfunction(connect)         # True
inspect.isclass(int)                  # True
inspect.ismethod(obj.method)         # True for bound methods
inspect.iscoroutinefunction(fn)        # True if async def
inspect.isgeneratorfunction(fn)        # True if uses yield
inspect.isbuiltin(len)               # True

getsource: read live source

pythonprint(inspect.getsource(connect))

# Works on classes, modules, and even lambdas
# (though lambdas are always on one line)
print(inspect.getsourcefile(connect))  # path to the file
print(inspect.getsourcelines(connect)) # (lines, start_lineno)

getmembers: explore a class or module

pythonimport pathlib

# All public methods of pathlib.Path
methods = [
    name
    for name, obj in inspect.getmembers(pathlib.Path)
    if inspect.isfunction(obj) and not name.startswith("_")
]

stack(): the call stack

pythonimport inspect

def who_called_me():
    frame_info = inspect.stack()[1]   # [0] is us, [1] is caller
    return f"{frame_info.filename}:{frame_info.lineno} in {frame_info.function}"

def outer():
    print(who_called_me())   # prints "script.py:N in outer"

inspect.stack() is how debuggers, profilers, and structured loggers attach call context to records. Expensive — don't call it in hot loops.

A practical use: auto-register handlers

pythonimport inspect, sys

def register_handlers(module):
    """Find all functions named handle_* in a module."""
    return {
        name: obj
        for name, obj in inspect.getmembers(module, inspect.isfunction)
        if name.startswith("handle_")
    }

handlers = register_handlers(sys.modules[__name__])

inspect.stack() acquires the GIL and walks all live frames. In async code, prefer traceback.format_stack() or structured logging with context vars (Day 088). getsource() reads disk — cache it if called frequently.

Run inspect.signature() on a FastAPI endpoint function. Look at the parameters. Notice how FastAPI's parameter injection maps to exactly what sig.parameters returns. Then read the FastAPI source for solve_dependencies — it's all inspect.signature calls.
