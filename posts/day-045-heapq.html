heapq: priority queues in the stdlib

Turn any list into a min-heap. nlargest and nsmallest find top-N in O(n log k) without sorting the whole list.

14 Feb 2026 • ~3 min read • Python 3.0+
TL;DR

heapq works on plain lists. heappush adds, heappop takes the smallest. For "top N" problems use nlargest(n, iterable, key=...) — faster than sorting and slicing when N is small relative to total length.

A heap is a tree where the smallest item is always at the root. Python's heapq uses a list as the backing store and gives you O(log n) push/pop. It's the right data structure for priority queues, top-K selections, and merging sorted streams.

The basic operations

pythonimport heapq

tasks = []
heapq.heappush(tasks, (3, "send report"))
heapq.heappush(tasks, (1, "page on-call"))
heapq.heappush(tasks, (2, "reply email"))

while tasks:
    priority, task = heapq.heappop(tasks)
    print(priority, task)
1 page on-call 2 reply email 3 send report

Tuples compare element-by-element, so (priority, task) sorts first by priority. The list itself is just a list — you can iterate it, check its length, copy it — but only push and pop through the heap functions to maintain the invariant.

Heapifying an existing list

Got an existing list? Don't push items one at a time. heapify rearranges in place in O(n). 

pythonitems = [3, 1, 4, 1, 5, 9, 2, 6]
heapq.heapify(items)
# items is now a valid heap; smallest is items[0]

The killer functions: nlargest, nsmallest

For "what are the 10 slowest devices?" or "top 5 errors by count?" — sorting the whole list is overkill. heapq.nlargest uses a heap of size N and runs in O(n log N). 

python# Top 5 slowest probes from a million
slowest = heapq.nlargest(5, probes, key=lambda p: p.latency)

# Three errors with the highest count
top3 = heapq.nlargest(3, counter.items(), key=lambda kv: kv[1])

For very small N relative to the data, nlargest beats sorted(...)[:N]. Around N == 10–20% of the data, plain sorted is competitive. Don't optimize until you've measured.

Merging sorted iterables

merge takes any number of sorted iterables and yields a single sorted stream. Lazy — doesn't materialize. 

python# Merging three sorted log files chronologically
def read_sorted(path):
    with open(path) as f:
        yield from f

for line in heapq.merge(
    read_sorted("region-us.log"),
    read_sorted("region-eu.log"),
    read_sorted("region-ap.log"),
):
    process(line)

Constant memory, regardless of file sizes. The classic external-sort building block.

The tie-breaker pattern

If your priority queue holds objects without a natural order and two priorities tie, the heap will compare the objects themselves — and crash if they don't support comparison. The fix: a counter as the tiebreaker. 

pythonimport heapq, itertools

counter = itertools.count()
queue   = []

def push(priority, job):
    heapq.heappush(queue, (priority, next(counter), job))

def pop():
    priority, _, job = heapq.heappop(queue)
    return job

The counter ensures each entry has a unique sort key, and as a side effect, equal-priority items come out in FIFO order — which is usually what you want anyway.

For a max-heap (largest on top), negate the priorities on push and pop. Python's heapq is deliberately min-only — the negation trick is idiomatic and adds no overhead.

Take a script that does sorted(items, key=...)[:N] for a small N over a large list. Replace it with heapq.nlargest(N, items, key=...). The measured speedup grows with the size of the input.
