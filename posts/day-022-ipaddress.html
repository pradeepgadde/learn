ipaddress: stop parsing IPs by hand

Subnet math, membership tests, and IPv4/IPv6 parity — all in the standard library. You should not be splitting strings on dots in 2026.

22 Jan 2026 • ~3 min read • Python 3.3+
TL;DR

The ipaddress module turns "10.0.0.0/24" into a real network object with subnet math, membership tests, and address iteration built in. Use ip_address, ip_network, and ip_interface — they handle IPv4 and IPv6 transparently.

Every networking script eventually grows a function called parse_cidr or ip_in_subnet. They're always slightly buggy, especially around IPv6. ipaddress made these obsolete in 2012 and most people still don't use it.

Three constructors, three jobs

pythonfrom ipaddress import ip_address, ip_network, ip_interface

# A single host address
addr = ip_address("192.168.1.10")

# A network (no host bits set)
net  = ip_network("192.168.1.0/24")

# An address with a prefix — the thing you'd see on `ip addr show`
iface = ip_interface("192.168.1.10/24")
print(iface.ip, iface.network)
# 192.168.1.10 192.168.1.0/24

IPv4 vs IPv6 is detected from the input. Need to force one? Use IPv4Network / IPv6Network explicitly — same API.

The killer move: membership tests

pythoncorp_net = ip_network("10.0.0.0/8")

ip_address("10.42.7.5") in corp_net      # True
ip_address("172.16.0.1") in corp_net     # False

# Network-in-network too
ip_network("10.1.0.0/16").subnet_of(corp_net)  # True
corp_net.overlaps(ip_network("10.1.0.0/16"))   # True

That in operator does the bit math for you. Works identically for IPv6.

Iterating addresses in a subnet

pythonnet = ip_network("192.168.1.0/29")   # 8 addresses

list(net.hosts())
# [IPv4Address('192.168.1.1'), ..., IPv4Address('192.168.1.6')]
# Excludes network and broadcast — the addresses you can actually assign

net.network_address     # 192.168.1.0
net.broadcast_address   # 192.168.1.7
net.netmask             # 255.255.255.248
net.prefixlen           # 29
net.num_addresses       # 8

For an IPv6 /64, hosts() returns a generator over 18 quintillion addresses. Don't materialize that.

Subnetting and supernetting

pythonnet = ip_network("10.0.0.0/22")

# Carve a /22 into four /24s
list(net.subnets(new_prefix=24))
# [10.0.0.0/24, 10.0.1.0/24, 10.0.2.0/24, 10.0.3.0/24]

# Walk up to a containing /16
net.supernet(new_prefix=16)
# 10.0.0.0/16

# Subtract — what's left of A after removing B?
list(ip_network("10.0.0.0/22").address_exclude(
    ip_network("10.0.1.0/24")))
# [10.0.0.0/24, 10.0.2.0/23] — minimal covering set

Useful classification properties

pythonaddr = ip_address("192.168.1.1")
addr.is_private       # True
addr.is_global        # False
addr.is_loopback      # False
addr.is_link_local    # False
addr.is_multicast     # False
addr.version          # 4

These work the same on IPv6 addresses. Filtering RFC1918 out of a log file becomes one line: [ip for ip in addrs if not ip.is_private].

ip_network("192.168.1.10/24") raises ValueError by default because host bits are set. Pass strict=False to accept it and have the host bits zeroed silently — useful when parsing interface configs straight from devices.

Pick an access-list, firewall rule, or routing table you have lying around. In ten lines, write a script that parses each CIDR with ip_network and prints which entries overlap with each other. The overlaps method does the work; you write the loop.
