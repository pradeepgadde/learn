enumerate, and the start= you forgot

The most-loved built-in has a forgotten parameter that does exactly what you keep doing manually.

05 Jan 2026 • ~3 min read • Python 3.0+
TL;DR

enumerate(items, start=1) gives you a 1-based index without any +1 arithmetic. Stop maintaining a manual counter. Combine with zip when you need the index across two parallel sequences.

Everyone knows enumerate. What most code reviews catch is that people still write the C-style loop next to it — a manual counter that increments at the bottom — or use range(len(things)) and then index back in. Both are giveaways.

The smell to delete

python# Manual counter — delete on sight
i = 0
for item in items:
    process(i, item)
    i += 1

# range(len()) anti-pattern — delete on sight
for i in range(len(items)):
    process(i, items[i])

# The right way
for i, item in enumerate(items):
    process(i, item)

The forgotten start=

For human-facing output — line numbers, "step 1 of N", ranked lists — you want 1-based indexing. Most people write i + 1 inside the loop. enumerate has a start parameter exactly for this. 

pythontasks = ["Wake up", "Make coffee", "Open editor", "Ship"]

for n, task in enumerate(tasks, start=1):
    print(f"{n}. {task}")
1. Wake up 2. Make coffee 3. Open editor 4. Ship

It works with any starting integer. Reading a file? Pass start=1 and your line numbers match what your editor shows you. 

pythonfor lineno, line in enumerate(file, start=1):
    if "TODO" in line:
        print(f"{path}:{lineno}: {line.strip()}")

enumerate over a zip

When you need an index and two parallel sequences, wrap a zip in enumerate. The unpacking nests cleanly. 

pythonnames  = ["Ada", "Linus", "Guido"]
scores = [88, 94, 99]

for rank, (name, score) in enumerate(zip(names, scores), start=1):
    print(f"#{rank}: {name:<6} {score}")
#1: Ada 88 #2: Linus 94 #3: Guido 99

On Python 3.10+, zip(a, b, strict=True) raises if the inputs aren't the same length — a free safety check that catches off-by-one bugs at the source.

Grep your codebase for range(len(. Almost every hit is a missed enumerate. Pick one and refactor it — the loop body usually loses an index expression and gets clearer.
