uv: pip and venv, but fast

One Rust binary that replaces pip, venv, pip-tools, and pyenv. The Python tooling consolidation has finally happened.

20 May 2026 • ~3 min read • Any Python
TL;DR

uv is a single binary that installs Python, creates virtualenvs, manages dependencies, and locks them — all 10–100× faster than the pip toolchain. Start with uv init, uv add <pkg>, uv run <script>.

The Python packaging conversation has been a mess for fifteen years: pip, pipenv, pip-tools, poetry, pdm, conda, pyenv. The uv team (the same folks behind ruff) collapsed all of it into one Rust-based command-line tool.

Install once, use everywhere

bash# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

No Python required to install uv — it's a standalone binary. It can then install any Python version it needs.

The five commands you'll use daily

bash# Start a new project
uv init my-app
cd my-app

# Add a dependency (also creates the venv on first call)
uv add httpx

# Add a dev-only dependency
uv add --dev pytest

# Run a script in the project's environment
uv run python main.py
uv run pytest

# Sync the environment to the lockfile (after pulling changes)
uv sync

That's the whole loop. uv init creates a pyproject.toml; uv add resolves and records the dependency in pyproject.toml and uv.lock; uv run guarantees you're running against the locked environment without you having to activate anything.

Speed: the part you have to see to believe

On a fresh machine, installing a typical web project's dependencies takes pip something like 30–60 seconds. uv does it in 1–3. Once the cache is warm, repeat installs are essentially instant. This isn't a marginal improvement — it changes how often you're willing to blow away and rebuild your environment.

It manages Python itself, too

No more pyenv, no more "which Python is python3?". Pin a Python version in pyproject.toml and uv installs and uses it. 

bash# Install a specific Python version
uv python install 3.12

# Pin this project to it
uv python pin 3.12

# See what's available
uv python list

Single-file scripts with inline metadata

For one-file utility scripts, you can declare dependencies inline using PEP 723. uv run picks them up, sets up an ephemeral environment, and runs the script. 

python# probe.py
# /// script
# requires-python = ">=3.12"
# dependencies = ["httpx", "rich"]
# ///

import httpx
from rich import print

print(httpx.get("https://api.github.com").json())
bashuv run probe.py     # Just works. No venv, no pip install.

For automation scripts that don't deserve a full project, this is transformative.

Already deep in poetry or pdm? You don't have to migrate. But for new projects — and for any context where install speed matters (CI, Docker images, Codespaces) — uv is the new default to reach for.

Install uv on your dev machine. Pick a small existing project. Inside it, run uv init --no-readme, then uv add each of its dependencies. Compare the time to install versus your current workflow.
