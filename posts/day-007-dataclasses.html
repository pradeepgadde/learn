dataclasses, with kw_only

One decorator replaces forty lines of class boilerplate. A handful of options turn it into a real configuration tool.

07 Jan 2026 • ~3 min read • Python 3.10+
TL;DR

@dataclass generates __init__, __repr__, and __eq__ from your type annotations. frozen=True makes instances immutable. kw_only=True forces callers to use keyword args, which is gold for any class with more than three fields.

Dataclasses arrived in Python 3.7 and have only gotten better. If you're still writing __init__ by hand to assign five attributes, you're doing manual labor a decorator was built to do for you.

The boilerplate it deletes

python# Before
class Point:
    def __init__(self, x, y, label=""):
        self.x = x
        self.y = y
        self.label = label
    def __repr__(self):
        return f"Point(x={self.x}, y={self.y}, label={self.label!r})"
    def __eq__(self, other):
        return (self.x, self.y, self.label) == (other.x, other.y, other.label)

# After
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float
    label: str = ""

Mutable defaults: use field

You can't write tags: list = [] — that default would be shared across instances, the classic Python gotcha. Dataclasses notice and raise. The fix is field(default_factory=list). 

pythonfrom dataclasses import dataclass, field

@dataclass
class Server:
    name: str
    tags: list[str] = field(default_factory=list)
    env:  dict[str, str] = field(default_factory=dict)

Immutable: frozen=True

Frozen dataclasses can't have their fields reassigned, and they become hashable — usable as dict keys or set members. Use this for value objects: configuration, coordinates, anything that represents a fact, not a state. 

python@dataclass(frozen=True)
class Coord:
    lat: float
    lon: float

visited = {Coord(37.77, -122.42), Coord(40.71, -74.00)}

The Python 3.10 upgrade: kw_only=True

Positional arguments are fine for two or three fields. Past that, callers start guessing the order. kw_only=True forces them to be explicit. 

python@dataclass(kw_only=True)
class RetryConfig:
    max_attempts: int = 3
    backoff:      float = 0.5
    jitter:       bool = True
    retry_on:     tuple = (ConnectionError,)

# Caller code is self-documenting:
cfg = RetryConfig(max_attempts=5, backoff=1.0)

You can also mark individual fields kw-only using field(kw_only=True) — useful when most fields are positional but a few are flag-like.

Need conversion or validation? Add a __post_init__(self) method — it runs after the generated __init__. For real schema validation though, reach for pydantic or attrs; dataclasses are deliberately minimal.

Find a class in your code with a hand-written __init__ that does nothing but assign attributes. Convert it to a dataclass. The diff will be satisfying.
