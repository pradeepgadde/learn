operator: itemgetter, attrgetter, methodcaller

Function forms of Python operators. itemgetter and attrgetter as fast, readable sort keys instead of lambdas.

4 Apr 2026 • ~2 min read • Python 3.8+
TL;DR

operator.itemgetter("key") is a faster, more composable alternative to lambda x: x["key"]. Same for attrgetter (attribute access) and methodcaller (method call). Combine with sorted(), min(), max(), and itertools.groupby().

itemgetter: dict and sequence keys

pythonfrom operator import itemgetter

devices = [
    {"hostname": "sw-03", "role": "core",   "uplinks": 4},
    {"hostname": "sw-01", "role": "access", "uplinks": 2},
    {"hostname": "sw-02", "role": "access", "uplinks": 2},
]

# Sort by role, then hostname
sorted(devices, key=itemgetter("role", "hostname"))

# Equivalent lambda (slower, less composable):
sorted(devices, key=lambda d: (d["role"], d["hostname"]))

When you pass multiple keys to itemgetter, it returns a tuple. Multi-key sorting in one call, no lambda.

attrgetter: object attributes

pythonfrom operator import attrgetter

devices = get_device_objects()

# Sort Device objects by .site.name then .hostname
sorted(devices, key=attrgetter("site.name", "hostname"))

# Dotted access works — reaches into nested attributes
get_name = attrgetter("site.name")
get_name(device)   # same as device.site.name

methodcaller: call a method on each item

pythonfrom operator import methodcaller

names = ["sw-Core-01", "sw-ACCESS-02", "rtr-Edge-01"]

# Lower-case all strings
list(map(methodcaller("lower"), names))
# ['sw-core-01', 'sw-access-02', 'rtr-edge-01']

# With arguments — strip a specific character
list(map(methodcaller("strip", "sw-"), names))

Pair with groupby

pythonfrom itertools import groupby
from operator  import itemgetter

by_role = itemgetter("role")
devices.sort(key=by_role)

for role, group in groupby(devices, key=by_role):
    print(role, list(group))

Operator functions for arithmetic

Every Python operator has a function form. Useful with functools.reduce or when you need to pass an operator as a callable. 

pythonimport operator, functools

# Sum a list via reduce (contrived but illustrative)
functools.reduce(operator.add, [1, 2, 3, 4])    # 10
functools.reduce(operator.mul, [1, 2, 3, 4])    # 24

# Available: add, sub, mul, truediv, floordiv, mod, pow
#            and_, or_, not_, lt, le, eq, ne, gt, ge
#            getitem, setitem, delitem, length_hint

itemgetter and attrgetter are implemented in C in CPython, so they're measurably faster than equivalent lambdas on large collections. For small lists the difference is noise; for data-processing pipelines over millions of records it adds up.

Find a sorted(..., key=lambda x: x["field"]) in your code. Replace with itemgetter("field"). Find a multi-key sort with a lambda returning a tuple. Replace with itemgetter("a", "b"). Each one is a small readability win and a minor speed win.
