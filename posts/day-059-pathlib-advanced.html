pathlib advanced: resolve, relative_to, walk

Beyond the basics — resolve absolute paths, escape symlinks, compute relative paths safely, and use Path.walk (3.12) for tree iteration.

28 Feb 2026 • ~3 min read • Python 3.12+
TL;DR

resolve() gives the absolute, symlink-followed path. relative_to(other) computes a relative path or raises if not possible. Path.walk() in 3.12 replaces os.walk with a pathlib-native version. is_relative_to is the safe "is this path inside that one" check.

Day 002 covered the basics — constructing paths, reading and writing, joining with /. Today: the operations that come up in real scripts and have non-obvious correct answers.

resolve: the canonical absolute path

pythonfrom pathlib import Path

p = Path("./logs/../logs/app.log")
p.resolve()
# PosixPath('/home/pradeep/projects/myapp/logs/app.log')

Three things resolve does at once: makes the path absolute, removes .. and . components, and follows symlinks. Compare to absolute(), which just prepends the cwd without normalizing — usually not what you want.

Pass strict=True to require the path to actually exist (raises FileNotFoundError if not). Useful when "this path must be real" is part of your precondition.

relative_to: the inverse of joining

pythonroot = Path("/srv/app")
file = Path("/srv/app/configs/devices/sw-01.yml")

file.relative_to(root)
# PosixPath('configs/devices/sw-01.yml')

# Raises ValueError if file isn't actually inside root
Path("/etc/passwd").relative_to(root)
# ValueError: '/etc/passwd' is not in the subpath of '/srv/app'

Useful when displaying paths to users, or when building URLs that mirror the directory tree.

The "is this path safe?" check

For any code that takes a user-supplied path and joins it with a base directory, you need to check the result didn't escape via ../../etc/passwd. Python 3.9 added is_relative_to for exactly this. 

pythonUPLOAD_DIR = Path("/srv/uploads").resolve()

def safe_join(user_input: str) -> Path:
    candidate = (UPLOAD_DIR / user_input).resolve()
    if not candidate.is_relative_to(UPLOAD_DIR):
        raise ValueError(f"path escape: {user_input}")
    return candidate

safe_join("image.png")               # OK
safe_join("../../etc/passwd")        # raises
safe_join("./safe/file.txt")         # OK

The resolve step is critical — without it, .. in the middle of the path doesn't get normalized. This pattern is one of the standard fixes for path-traversal vulnerabilities.

Path.walk: the 3.12 upgrade to os.walk

python# Python 3.12+
for dirpath, dirnames, filenames in Path(".").walk():
    # dirpath is a Path; dirnames and filenames are lists of strings
    # Mutating dirnames in-place prunes the walk (same as os.walk)
    dirnames[:] = [d for d in dirnames if not d.startswith(".")]
    for name in filenames:
        file = dirpath / name
        # do something with file

Cleaner than os.walk because the first element is a real Path object, not a string — chain pathlib calls directly. The ability to prune with dirnames[:] = ... is unchanged.

On Python 3.11 or older, stick with os.walk or Path.rglob. rglob is nicer for "find files matching a pattern," walk is nicer for "visit every directory and decide what to do per dir."

Useful one-liners

python# Get the parts of a path as a tuple
Path("/etc/nginx/sites-enabled/default").parts
# ('/', 'etc', 'nginx', 'sites-enabled', 'default')

# Stem (filename without extension), suffix (extension with dot)
Path("backup.tar.gz").stem      # 'backup.tar'
Path("backup.tar.gz").suffix    # '.gz'
Path("backup.tar.gz").suffixes  # ['.tar', '.gz']

# Replace just the suffix
Path("report.md").with_suffix(".html")    # PosixPath('report.html')

# Replace just the filename
Path("/etc/nginx/old.conf").with_name("new.conf")
# PosixPath('/etc/nginx/new.conf')

# Read multi-line file as a list
Path("hosts.txt").read_text().splitlines()

Mkdir patterns

python# The "mkdir -p" equivalent
Path("deeply/nested/dir").mkdir(parents=True, exist_ok=True)
# Creates all parents, doesn't error if it already exists

Both flags off (the default) is rarely what you want. parents=True, exist_ok=True is the form you want roughly 95% of the time.

One closing tip: Path("~/.config/myapp") .expanduser() turns ~ into the actual home directory. Without it, you'll be looking for a literal ~ directory.

Audit any code that takes a user-supplied filename and joins it with a base directory. Add the safe_join pattern from above. If you'd never written that check, you're not alone — and now your code handles a whole class of attack it didn't before.
