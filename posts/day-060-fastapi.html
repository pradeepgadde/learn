FastAPI: an API in twelve lines

Type hints become a validated, documented HTTP API. Path operations, request bodies, dependencies — all from the function signature.

01 Mar 2026 • ~3 min read • Python 3.8+
TL;DR

Decorate a function with @app.get("/path"), annotate parameters with types and pydantic models, run with uvicorn app:app --reload. You get validation, JSON parsing, OpenAPI schema, and interactive docs at /docs for free.

FastAPI is what happens when typer (Day 050) meets the web. The same design philosophy — the function signature is the schema — produces a server that validates input, generates docs, and runs fast enough to compete with Go and Node.

The minimum

python# app.py
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Device(BaseModel):
    hostname: str
    role:     str
    port:     int = 22

@app.get("/health")
def health():
    return {"ok": True}

@app.post("/devices")
def create_device(d: Device) -> Device:
    return d
bashuv add fastapi uvicorn
uvicorn app:app --reload
# Browse http://localhost:8000/docs for live, interactive Swagger UI

The Device model validates the request body. The return type drives the response schema. Hit /docs and you get Swagger UI with a try-it-out button. Hit /redoc for the ReDoc version. None of this required configuration.

Path and query parameters

python@app.get("/devices/{hostname}")
def get_device(hostname: str, verbose: bool = False):
    # /devices/sw-01?verbose=true
    return {"hostname": hostname, "verbose": verbose}

Parameters in the path go into URL placeholders. Everything else becomes a query parameter (with validation). ?verbose=true is parsed to True; ?verbose=banana returns a 422 with a clear error.

Response model: separate input from output

The same model rarely fits both directions — you don't want to accept an id on creation but you want to return one. Use response_model to pin the output shape. 

pythonclass DeviceIn(BaseModel):
    hostname: str
    role:     str

class DeviceOut(DeviceIn):
    id:        int
    created_at: datetime

@app.post("/devices", response_model=DeviceOut)

Anything not in DeviceOut (passwords, internal tokens, raw DB columns) gets stripped from the response. Cheap data-leakage defense.

Dependencies: shared setup per request

Dependencies are functions whose result gets injected. Use them for auth, DB sessions, common query parameters, anything per-request. 

pythonfrom fastapi import Depends, HTTPException, Header

def require_admin(x_token: str = Header(...)):
    if x_token != "secret":
        raise HTTPException(status_code=401, detail="unauthorized")
    return "admin"

@app.delete("/devices/{hostname}")
def delete_device(hostname: str, user: str = Depends(require_admin)):
    ...

The dependency runs before the path function. Its return value becomes the parameter value. Failures short-circuit with the right HTTP status.

Async by default

Most path functions can be async def. Combined with httpx (Day 017) or asyncssh (Day 025), a single FastAPI worker can handle hundreds of concurrent slow requests. 

pythonimport httpx

@app.get("/probe/{host}")
async def probe(host: str):
    async with httpx.AsyncClient() as c:
        r = await c.get(f"https://{host}", timeout=5)
    return {"status": r.status_code}

Mix and match async and sync handlers freely — FastAPI runs sync ones in a thread pool so they don't block the event loop.

FastAPI vs Flask in 2026. Flask is still fine for tiny services and templated HTML apps. For new JSON APIs, FastAPI's free validation, free docs, and async-first design are hard to argue with. Migration from Flask is straightforward — both use Python functions as routes.

Build a tiny FastAPI app with one POST endpoint that takes a pydantic model. Run it. Open /docs in your browser. Submit a request with the wrong type in one field and watch the error response point at exactly the broken field. You'll feel why people ditch Flask for it.
