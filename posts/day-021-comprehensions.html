Comprehensions you'll actually read later

List comprehensions are Python's most-loved feature and its most-abused. Three rules keep them on the right side of clever.

21 Jan 2026 • ~3 min read • Python 3.0+
TL;DR

List, dict, and set comprehensions are pythonic when they fit on one readable line and have at most one filter and one nesting level. Past that, write a regular loop. The comprehension is for clarity, not for showing off.

A list comprehension says "build a list by transforming these inputs." A dict comprehension says the same about a dict. A set comprehension, a set. The mistake is treating them as a contest in concision — the goal is a single line that reads like a sentence.

The three flavors

python# List
squares      = [x * x for x in range(10)]

# Set — uniqueness for free
unique_tags  = {t.lower() for item in items for t in item.tags}

# Dict — keyed transform
by_id        = {u.id: u for u in users}

# Generator — same syntax with parentheses (lazy; see Day 020)
lengths      = (len(w) for w in words)

Filtering and the conditional

An if at the end filters. An if/else earlier transforms based on a condition. These look similar — they're not. 

python# Filter: only positives are included
pos = [x for x in values if x > 0]

# Transform: every value, but negatives become 0
clamped = [x if x > 0 else 0 for x in values]

# Both — filter, then transform
scaled = [x * 2 for x in values if x > 0]

The line where it stops being clever

These two are clear: 

pythontitles = [b.title for b in books]
cheap  = [b for b in books if b.price < 10]

This is borderline:

pythondiscounted_titles = [
    b.title.upper() for b in books
    if b.price < 10 and b.in_stock
]

And this is where you write a real loop:

python# DON'T — clever to write, awful to read in three months
flattened = [
    (k, v) for section in config.values()
    if isinstance(section, dict)
    for k, v in section.items()
    if not k.startswith("_")
]

# DO — boring, obvious, modifiable
flattened = []
for section in config.values():
    if not isinstance(section, dict):
        continue
    for k, v in section.items():
        if not k.startswith("_"):
            flattened.append((k, v))

My rule of thumb: a comprehension that needs more than one for or more than one if is begging to be a regular loop. The diff is two more lines and an order of magnitude more readable.

Three useful idioms worth keeping in your back pocket

Lookup dict from a list of objects:

pythonby_id = {u.id: u for u in users}

Inverting a dict (when values are unique):

pythonname_by_id = {v: k for k, v in id_by_name.items()}

Deduping while preserving order (Python 3.7+):

pythonunique = list(dict.fromkeys(items))

Comprehensions create their own scope in Python 3. The loop variable doesn't leak out, which is good news. The flip side: variables defined inside don't survive the comprehension — if you need the running value afterward, use a regular loop.

Find a comprehension in your codebase that's longer than one line. Time how long it takes you to figure out what it does. If it's more than ten seconds, rewrite it as a regular loop and feel the relief.
