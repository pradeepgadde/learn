deque: lists with fast left-side ops

O(1) appends and pops on both ends. A maxlen for bounded ring buffers. The right choice for queues, rolling windows, and recent-N caches.

16 Feb 2026 • ~3 min read • Python 3.0+
TL;DR

collections.deque is a doubly-linked list with O(1) append, appendleft, pop, and popleft. maxlen creates a bounded ring buffer that drops the oldest element when full. Use for queues (FIFO) and recent-N tracking; use a list for everything else.

Lists are great until you start operating on the front. list.pop(0) and list.insert(0, x) are both O(n) — every element shifts. For queues and sliding windows, you want a deque.

The basic operations

pythonfrom collections import deque

q = deque(["a", "b", "c"])

q.append("d")         # right side
q.appendleft("z")     # left side
# deque(['z', 'a', 'b', 'c', 'd'])

q.pop()               # right — 'd'
q.popleft()           # left — 'z'

Same indexing and iteration as a list. Same len(). The only thing you give up is fast middle access — q[len(q)//2] is O(n) on a deque, O(1) on a list.

The killer feature: maxlen

Pass a maximum length and the deque becomes a ring buffer — appending past the limit silently drops the oldest element from the other end. 

python# Keep the last 100 log lines for diagnostics
recent = deque(maxlen=100)

for line in log:
    recent.append(line)

# 'recent' always holds at most 100 lines.
# When the 101st arrives, the first one drops automatically.

No bookkeeping, no indexing math, no off-by-one bugs. The right shape for "remember the last N of anything."

Sliding-window aggregations

Combine maxlen with the math you want and you have a moving window. For a moving average: 

pythonwindow     = deque(maxlen=10)
moving_avg = []

for val in samples:
    window.append(val)
    moving_avg.append(sum(window) / len(window))

For huge windows where recomputing sum(window) per sample matters, keep a running sum: add the new value, subtract the one being evicted (track it before the append).

BFS without the smell

Breadth-first search over a graph — classic deque territory. 

pythondef bfs(graph, start):
    visited = {start}
    queue   = deque([start])
    while queue:
        node = queue.popleft()
        yield node
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

With a list, that queue.pop(0) would shift every other element each iteration — O(n²) total. Deque keeps it O(n).

Rotation, in case you ever need it

rotate(n) shifts elements right by n (negative for left). Useful for round-robin scheduling. 

pythonworkers = deque(["alice", "bob", "carol", "dan"])

def next_worker():
    w = workers[0]
    workers.rotate(-1)         # shift left by one
    return w

Thread-safety, briefly

append/popleft on a deque are atomic at the bytecode level — safe for single-producer, single-consumer patterns without a lock. For multi-producer or multi-consumer queues, use queue.Queue (from the queue module) instead, which has proper synchronization.

For asyncio queues across coroutines, use asyncio.Queue. Same FIFO API, but put and get are awaitable, so producers and consumers can suspend on a full or empty queue without blocking the loop.

If you have any code that does list.pop(0) in a loop — a worker queue, a parsing front-of-list pattern, anything FIFO — switch to a deque. Same API, dramatically better scaling.
