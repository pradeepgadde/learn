Logging that doesn't suck

logging has a bad reputation because the docs lead with the hard part. The easy version is six lines and worth every minute.

17 May 2026 • ~3 min read • Python 3.0+
TL;DR

Call logging.basicConfig() once at the top of your entry point. In every module, do log = logging.getLogger(__name__). Use log.info(), log.warning(), log.error() instead of print. You get levels, timestamps, and routing for free.

Print statements are fine for one-off scripts. The moment something runs unattended — a cron job, a service, a long-running CLI — you want timestamps, levels, and the ability to silence noise without editing source. That's what logging does.

The minimal setup

python# In your entry-point script (the one with `if __name__ == "__main__":`)
import logging

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s %(levelname)-8s %(name)s: %(message)s",
    datefmt="%H:%M:%S",
)

log = logging.getLogger(__name__)
log.info("Starting up")
14:03:21 INFO __main__: Starting up

Six lines, and you've replaced print with something you'll never need to rip out.

The five levels, and when to use each

  • debug — detail you only want when something's wrong. Variable values, branch traces.
  • info — "this happened." Connection opened, file processed, request handled.
  • warning — "this is unusual but I handled it." Retry succeeded, deprecated config used.
  • error — "this operation failed but the program continues." Caught exceptions, partial failures.
  • critical — "the program can't continue." Use sparingly.

Want everything? level=logging.DEBUG. Want only problems? level=logging.WARNING. One line, no code change to switch.

The cardinal rule: use __name__

logging.getLogger(__name__) in every module gives you a per-module logger. The name shows up in every log line, so you instantly know where a message came from. 

python# in app/network/probe.py
import logging
log = logging.getLogger(__name__)

def ping(host):
    log.info("pinging %s", host)
    ...
14:03:21 INFO app.network.probe: pinging router1

Bonus: you can now silence one chatty module without touching the others: logging.getLogger("app.network.probe").setLevel("WARNING").

Lazy formatting matters

Pass arguments to the logger; don't pre-format them. If the message is below the threshold and would be discarded, the formatting is skipped entirely. 

python# Don't — formats the f-string even when DEBUG is off
log.debug(f"processed {len(items)} items in {elapsed:.2f}s")

# Do — % formatting is deferred to the logging machinery
log.debug("processed %d items in %.2fs", len(items), elapsed)

For most code this is a micro-optimization. For tight loops or DEBUG-level logs that gather a lot of context, it matters.

Logging exceptions

Inside an except block, use log.exception — it logs the message at ERROR level and includes the full traceback automatically. 

pythontry:
    connect(host)
except ConnectionError:
    log.exception("failed to connect to %s", host)
    # Don't re-raise unless you actually want to crash

For structured logging (JSON output to ship to Loki, ELK, Splunk), look at structlog. For more readable terminal output during development, look at rich.logging. Both layer cleanly on top of the standard logging module.

Open a script you've written. Replace every print that's a diagnostic (not a CLI output) with the appropriate log.info / log.warning / log.error. Add the basicConfig at the top. Now you can run it with --verbose support next week without rewriting anything.
