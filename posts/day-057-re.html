re: regex features past the basics

Named groups, verbose mode for readable patterns, lookahead and lookbehind. The features that turn regex from write-only to maintainable.

26 Feb 2026 • ~3 min read • Python 3.0+
TL;DR

Use (?P<name>...) for named groups. Use re.VERBOSE to write commented, multi-line patterns. re.compile once at module level for anything you'll match repeatedly. For complex text extraction, parse-don't-regex.

Regex earns its reputation for being unreadable. Three Python features turn most patterns into something you can come back to a year later without confusion.

Named groups

Numbered groups (\1, match.group(2)) are fragile — renumbering on edit is a real risk. Named groups make the intent explicit. 

pythonimport re

pattern = re.compile(r"(?P<ip>\d+\.\d+\.\d+\.\d+) - - "
                     r"\[(?P<ts>[^\]]+)\] "
                     r'"(?P<method>\w+) (?P<path>\S+) HTTP/[\d.]+" '
                     r'(?P<status>\d+) (?P<size>\d+)')

m = pattern.match(log_line)
if m:
    m["ip"], m["path"], m["status"]
    # Or m.groupdict() for the whole bundle as a dict

Reading the result line is much closer to reading natural language. Adding a new group in the middle doesn't break later references.

Verbose mode for readable patterns

With re.VERBOSE, whitespace inside the pattern is ignored and # starts a comment. Suddenly you can format a regex like code. 

pythonphone_re = re.compile(r"""
    (?P<country>\+\d{1,3})?     # optional +CC country code
    \s*
    \(?(?P<area>\d{3})\)?        # area code, optionally parenthesised
    [-.\s]*                       # any separator
    (?P<exchange>\d{3})           # exchange
    [-.\s]*
    (?P<line>\d{4})              # line
""", re.VERBOSE)

That's the same pattern that, packed into one line, would be unreadable. The only catch: to match a literal space or hash, escape them (\ , \#) or use a character class ([ ], [#]).

Compile once, use many times

For patterns used in a loop or hot path, compile them once at module level. Python caches recent patterns internally, but explicit compilation is faster and clearer about intent. 

python# Module level — compiled once at import
_LOG_RE = re.compile(r"^(?P<level>\w+):\s+(?P<msg>.*)$")

def parse(line):
    m = _LOG_RE.match(line)
    return m.groupdict() if m else None

Lookahead and lookbehind

Match positions that have certain context, without consuming the context. 

python# 'foo' but only when followed by '.bar' (don't capture .bar)
re.findall(r"foo(?=\.bar)", "foo.bar foo.baz")
# ['foo']

# 'foo' but NOT when followed by '.bar'
re.findall(r"foo(?!\.bar)", "foo.bar foo.baz")
# ['foo'] (the second one)

# 'bar' but only when preceded by 'foo.'
re.findall(r"(?<=foo\.)bar", "foo.bar baz.bar")
# ['bar'] (the first one)

Useful for filtering matches by surrounding context without including the context in the result.

Substitution with backrefs and functions

python# Backref by group number
re.sub(r"(\w+)\.(\w+)", r"\2.\1", "a.b c.d")
# 'b.a d.c'

# Backref by name
re.sub(r"(?P<first>\w+) (?P<last>\w+)",
       r"\g<last>, \g<first>",
       "Pradeep Gadde")

# Or pass a function for full computation
re.sub(r"\d+", lambda m: str(int(m[0]) * 2), "a 1 b 2 c 3")
# 'a 2 b 4 c 6'

The catastrophic-backtracking trap

Patterns with nested quantifiers ((a+)+, (.*)*) can explode on certain inputs — runtime grows exponentially with input length. A common production crash. 

python# Looks innocent, can take seconds on a 30-char input
re.match(r"(a+)+b", "a" * 30)

# Better: avoid nesting quantifiers, or use a possessive/atomic
# group. Python doesn't support possessive groups, so be careful with structure.
re.match(r"a+b", "a" * 30)   # fast — single quantifier

When to skip regex

Regex is wrong for nested or recursive structures — HTML, JSON, programming languages, structured logs with embedded objects. For those, use a real parser: html.parser, json.loads, dedicated libraries for the format. Network output specifically: TextFSM (Day 028) is purpose-built.

For interactive regex development, paste your pattern into regex101 with the Python flavor selected. The visual breakdown of how each token works has saved countless hours of head-scratching.

Take a regex with two or more numbered groups in your code. Convert to named groups. Use m.groupdict() to pull the values out by name. The next person to edit the pattern (you in six months) will thank you.
