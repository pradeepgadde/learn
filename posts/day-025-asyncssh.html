asyncssh: poll 100 devices in parallel

Paramiko, but on asyncio. Hundreds of SSH sessions on one thread, in roughly the time of one.

25 Jan 2026 • ~3 min read • Python 3.8+
TL;DR

asyncssh gives you a clean async/await API for SSH. Combined with asyncio.gather and a Semaphore for backpressure, you can run a command across an entire device fleet in seconds. Use this when netmiko's threaded approach starts straining.

Threads work for tens of devices. For hundreds, the overhead and the GIL contention start to bite, and your script becomes slower than the actual SSH round-trips justify. Async I/O is what you want here — SSH is purely network-bound.

The minimal async SSH call

pythonimport asyncio, asyncssh

async def run_on(host, cmd):
    async with asyncssh.connect(
        host, username="netadmin", known_hosts=None,
    ) as conn:
        result = await conn.run(cmd, check=True)
        return host, result.stdout

host, out = asyncio.run(run_on("core-sw-01", "show version"))

Same shape as a paramiko call, but every I/O line is awaited, so the event loop can interleave other connections while this one is waiting on the network.

Fanning out across the fleet

pythonasync def poll_all(hosts, cmd):
    results = await asyncio.gather(
        *(run_on(h, cmd) for h in hosts),
        return_exceptions=True,
    )
    for r in results:
        if isinstance(r, Exception):
            print(f"FAILED: {r}")
        else:
            host, out = r
            print(f"--- {host} ---\n{out}")

asyncio.run(poll_all(["sw-01", "sw-02", "sw-03"], "show clock"))

return_exceptions=True is critical for fleet work. Without it, one unreachable device crashes the entire gather. With it, exceptions become return values you can report on.

Bounding concurrency with a Semaphore

Launching 500 concurrent SSH sessions will exhaust your file descriptors and probably overwhelm your auth server. Cap it with a semaphore. 

pythonasync def poll_bounded(hosts, cmd, limit=25):
    sem = asyncio.Semaphore(limit)

    async def guarded(host):
        async with sem:
            return await run_on(host, cmd)

    return await asyncio.gather(
        *(guarded(h) for h in hosts),
        return_exceptions=True,
    )

At any moment, at most 25 SSH sessions are active. As one finishes, the next one starts. Your TACACS server will thank you.

Per-call timeouts

Each SSH call should have a timeout, and the whole batch should too. asyncssh accepts connect_timeout on the connect call; wrap the run in asyncio.wait_for to bound the command itself. 

pythonasync def run_on(host, cmd):
    async with asyncssh.connect(
        host, username="netadmin", known_hosts=None,
        connect_timeout=5,
    ) as conn:
        result = await asyncio.wait_for(
            conn.run(cmd, check=True),
            timeout=15,
        )
        return host, result.stdout

SCP and SFTP, while you're here

asyncssh handles file transfer too — same connection, same async pattern. Useful for grabbing config backups or pushing image files. 

pythonasync with asyncssh.connect("sw-01", username="netadmin") as conn:
    await asyncssh.scp((conn, "flash:running-config"), "backups/sw-01.cfg")

known_hosts=None disables host-key checking. Fine for quick scripts in a controlled lab, dangerous in production. For fleet work, point at your real known_hosts file: known_hosts="~/.ssh/known_hosts".
