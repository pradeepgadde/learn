match isn't switch

Treat match as a value-comparison and you'll be unimpressed. Treat it as shape-comparison and it replaces a chapter of code.

12 May 2026 • ~3 min read • Python 3.10+
TL;DR

match is structural pattern matching, not a switch. Patterns can deconstruct lists, dicts, and class instances and bind their parts to variables — in one step, with no nested if chains.

When match arrived in Python 3.10, half the community shrugged: "we already have if." That misses the point. Pattern matching isn't about choosing branches by value — it's about choosing branches by shape, and binding the pieces while you do it.

The boring case (where switch would do)

pythonmatch status:
    case 200:        handle_ok()
    case 404:        handle_missing()
    case 500 | 502 | 503:  handle_server_error()
    case _:          handle_other()

Fine. Cleaner than a long if/elif. But this is the least interesting use of the feature.

Sequence patterns

A pattern that looks like a list matches one. The names inside get bound to the elements. *rest works as you'd expect. 

pythondef describe(items):
    match items:
        case []:
            return "empty"
        case [x]:
            return f"singleton: {x}"
        case [first, *rest]:
            return f"head {first}, then {len(rest)} more"

describe([1, 2, 3])    # 'head 1, then 2 more'

Mapping patterns

Match a dict's shape — required keys, ignored extras — and bind the values. Replaces a forest of if "x" in d and "y" in d. 

pythondef handle_event(event):
    match event:
        case {"type": "login", "user": user}:
            audit(user)
        case {"type": "error", "code": code, "msg": msg}:
            log_error(code, msg)
        case {"type": t}:
            log(f"unhandled event: {t}")

Each case requires those exact keys to be present, but doesn't care about extras. The matched values flow straight into the body.

Class patterns

This is where match earns its keep. You can match against a class and destructure its attributes in one step. 

pythonfrom dataclasses import dataclass

@dataclass
class Circle: radius: float

@dataclass
class Rect:   w: float; h: float

def area(shape):
    match shape:
        case Circle(radius=r):
            return 3.14159 * r * r
        case Rect(w=w, h=h):
            return w * h

Guards

Add an if after a pattern to constrain it further. The pattern matches structure; the guard adds arbitrary logic. 

pythonmatch point:
    case (x, y) if x == y:
        print("on the diagonal")
    case (x, y) if x * x + y * y < 1:
        print("inside unit circle")
    case _:
        print("elsewhere")

Trap: a bare name like case x: is a capture, not a comparison — it always matches and binds the value to x. To compare against a constant, use a dotted name (case Color.RED:) or wrap in a literal pattern.

Find a long if/elif chain in your code that inspects the structure of a dict or object — webhook handlers and event dispatchers are good hunting grounds. Rewrite it as a match. Watch the indentation flatten.
