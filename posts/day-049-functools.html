functools: partial, reduce, and friends

partial bakes in arguments. reduce folds a sequence. wraps keeps decorators honest. Three small tools that pay compound interest.

18 Feb 2026 • ~3 min read • Python 3.0+
TL;DR

partial(fn, *args, **kw) returns a new callable with some arguments pre-filled. reduce(fn, iterable, init) applies a binary function to fold a sequence into a single value. @wraps(fn) keeps a decorator from eating the name and docstring of the function it wraps.

We covered functools.cache on Day 008 — the most-used member of the module. The next three you'll reach for almost as often.

partial: bake in some arguments

Take a function with several parameters, lock in some of them, and pass the simpler signature elsewhere. Cleaner than a lambda, more introspectable. 

pythonfrom functools import partial
import httpx

# Build a one-arg fetcher pre-configured with timeout and headers
fetch = partial(httpx.get,
                timeout=5,
                headers={"User-Agent": "probe/1.0"})

r = fetch("https://example.com")

Particularly nice with map, callbacks, and thread/process pools (Day 038) where you need a callable that takes one argument: 

pythonfrom concurrent.futures import ThreadPoolExecutor

def probe(host, port, timeout=3): ...

# Pre-fill port and timeout, fan out across hosts
with ThreadPoolExecutor() as ex:
    results = ex.map(partial(probe, port=443, timeout=5), hosts)

reduce: fold a sequence to a single value

Apply a two-argument function across a sequence, accumulating. The "compose many things into one" operation. Often replaceable by sum, min, max, or any/all — reach for reduce only when none of those fit. 

pythonfrom functools import reduce
import operator

# Multiply all numbers (no built-in for this until math.prod in 3.8)
reduce(operator.mul, [2, 3, 4])           # 24

# Merge a list of dicts (later wins)
configs = [{"a": 1}, {"b": 2}, {"a": 99}]
merged  = reduce(lambda acc, d: {**acc, **d}, configs, {})

# Bitmask combine
flags = reduce(operator.or_, [0b001, 0b010, 0b100])    # 0b111

The third argument is the initial value — pass it explicitly to handle empty sequences without crashing.

wraps: keep decorators honest

Without @wraps, every decorated function loses its name and docstring — bad for debugging, bad for help(), bad for any tool that introspects. 

pythonfrom functools import wraps
import time

def timed(fn):
    @wraps(fn)
    def inner(*a, **kw):
        t0 = time.perf_counter()
        try:
            return fn(*a, **kw)
        finally:
            print(f"{fn.__name__}: {(time.perf_counter() - t0) * 1000:.1f}ms")
    return inner

@timed
def slow_thing():
    """Useful diagnostics tool."""
    ...

slow_thing.__name__      # 'slow_thing' (with @wraps); 'inner' (without)
slow_thing.__doc__       # 'Useful diagnostics tool.'

One line in the decorator definition, every decorated function keeps its identity. There is no good reason not to use it.

singledispatch: poor man's multimethods

For functions that should behave differently depending on the type of the first argument — without writing a class hierarchy. 

pythonfrom functools import singledispatch

@singledispatch
def to_json(obj):
    raise TypeError(obj)

@to_json.register
def _(obj: datetime): return obj.isoformat()

@to_json.register
def _(obj: Path):     return str(obj)

@to_json.register
def _(obj: set):      return sorted(obj)

Type-based dispatch in pure Python, no runtime isinstance checks. Pairs nicely with the JSON default= hook from Day 015.

A few honourable mentions: functools.cmp_to_key for adapting old-style comparison functions to the modern key= API; cached_property for lazy attributes; total_ordering to fill in comparison operators given just one or two.

Find a lambda in your codebase that just pre-fills arguments to another function. Replace with partial. Stack-traces and repr output get nicer; behavior is identical.
