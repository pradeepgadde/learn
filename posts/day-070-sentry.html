Sentry: errors with context

Unhandled exceptions with full tracebacks, request context, and breadcrumbs. Five lines of init replace your custom error reporting.

11 Mar 2026 • ~2 min read • Python 3.7+
TL;DR

Call sentry_sdk.init(dsn="...", traces_sample_rate=0.1) at startup. Unhandled exceptions are captured automatically. Add custom context with sentry_sdk.set_tag, set_user, set_context. Capture handled errors deliberately with capture_exception.

Logging that an error happened is necessary but not sufficient. Sentry de-duplicates errors, groups by stack trace, surfaces the new ones, attaches request data, and links to git commits. It's the difference between "we got 4000 ERROR-level log lines" and "this one bug fired 4000 times in the last hour, here are the affected users."

The minimal init

pythonimport sentry_sdk

sentry_sdk.init(
    dsn=os.environ["SENTRY_DSN"],
    environment=os.environ.get("ENV", "dev"),
    release=os.environ.get("GIT_SHA"),
    traces_sample_rate=0.1,           # 10% of transactions traced
    profiles_sample_rate=0.1,         # 10% of those also profiled
)

Three things matter here. environment separates dev from staging from prod in the Sentry UI; release ties errors to the commit that introduced them; sampling rates keep traffic manageable on busy services.

Framework integrations

Sentry auto-detects most popular frameworks. For FastAPI, Flask, Django, and Celery, no extra setup is needed beyond init. For others, pass an integration explicitly. 

pythonfrom sentry_sdk.integrations.fastapi import FastApiIntegration
from sentry_sdk.integrations.sqlalchemy import SqlalchemyIntegration

sentry_sdk.init(
    dsn=...,
    integrations=[
        FastApiIntegration(),
        SqlalchemyIntegration(),
    ],
)

With the FastAPI integration, request method/path/headers are auto-attached to every captured error. With SQLAlchemy, the slow query that triggered the error appears in the trace.

Adding context manually

python# Per-request user context
sentry_sdk.set_user({"id": user.id, "email": user.email})

# Tag for filtering — kept low-cardinality
sentry_sdk.set_tag("deploy_region", "us-east-1")

# Arbitrary structured context — not searchable but visible per error
sentry_sdk.set_context("device", {
    "hostname": device.hostname,
    "role":     device.role,
    "vendor":   device.vendor,
})

Capturing handled errors

Some errors are caught and you want to recover from them — but you still want Sentry to know. Capture explicitly. 

pythontry:
    push_config(device)
except DeviceTimeout as e:
    log.warning("timeout, will retry", device=device.hostname)
    sentry_sdk.capture_exception(e)
    retry_later(device)

For non-exception events (like "configuration drift detected"): sentry_sdk.capture_message("...", level="warning").

before_send: redact sensitive data

Before any event leaves the process, you can modify or drop it. Use this to scrub secrets from headers, mask PII, or filter noisy third-party errors. 

pythondef before_send(event, hint):
    # Drop boring asyncio.CancelledError
    if exc_info := hint.get("exc_info"):
        if isinstance(exc_info[1], asyncio.CancelledError):
            return None
    # Redact Authorization headers
    request = event.get("request", {})
    request.get("headers", {}).pop("Authorization", None)
    return event

sentry_sdk.init(dsn=..., before_send=before_send)

Self-hosted vs SaaS

Sentry has both. The SaaS is faster to start and well-priced for small teams; the self-hosted version (Docker Compose stack) keeps everything on your network. The Python SDK works identically against either.

For OSS alternatives, look at GlitchTip (Sentry-API-compatible, lighter weight) or OpenTelemetry's "errors as span events" model (Day 068). For most teams, paying Sentry is worth not maintaining yet another backend.

Add Sentry to a service. Trigger an intentional 1/0 in some unimportant endpoint. Watch the error appear in the dashboard with the full request, headers, breadcrumbs, and a link to the exact source line. The first time you see this you stop tolerating bare except Exception: pass blocks.
