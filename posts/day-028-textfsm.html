TextFSM: parse CLI output sanely

Stop writing brittle regex pipelines for show commands. TextFSM is a state machine for text, and somebody else has already written most of the templates.

28 Jan 2026 • ~3 min read • Python 3.7+
TL;DR

TextFSM templates describe states and rules for parsing text. ntc-templates already ships templates for thousands of show commands across Cisco, Juniper, Arista, and others. Use the textfsm-clean, get_structured_data helper, or netmiko's built-in use_textfsm=True — all three reach the same library.

Network device output is human-friendly text in fixed-width columns, and parsing it with regex is a known recipe for sadness. TextFSM, originally from Google, is a domain language for "left-aligned table with these columns, possibly multi-line records." Once you've used it, the regex approach feels primitive.

The library you don't have to write

Before writing your own template, check ntc-templates. It's the de-facto open library for network CLI parsing — thousands of templates for Cisco, Juniper, Arista, Palo Alto, and dozens more. Maintained by the community, fast to update. 

bashpip install ntc-templates textfsm

Parse output you already have

pythonfrom ntc_templates.parse import parse_output

raw = """\
Interface              IP-Address      OK? Method Status                Protocol
GigabitEthernet0/0     10.0.0.1        YES manual up                    up
GigabitEthernet0/1     unassigned      YES NVRAM  administratively down down
"""

records = parse_output(
    platform="cisco_ios",
    command="show ip interface brief",
    data=raw,
)
for r in records:
    print(r["interface"], r["ipaddr"], r["status"])
GigabitEthernet0/0 10.0.0.1 up GigabitEthernet0/1 unassigned administratively down

That's the whole interaction with the library — platform name, command name, raw text. Output is a list[dict[str, str]] ready to filter or shove into your monitoring system.

Through netmiko, in one call

If you're already shelling into devices with netmiko (Day 024), pass use_textfsm=True and skip the intermediate text entirely. 

pythoninterfaces = net.send_command(

# Already a list of dicts

up_with_ip = [

Writing your own template (when ntc-templates doesn't have it)

The format is small. States, value declarations, regex rules with named captures pointing at value names. 

textfsmValue INTERFACE (\S+)
Value STATE (up|down|admin-down)
Value SPEED (\S+)

Start
  ^${INTERFACE}\s+is\s+${STATE},\s+line protocol -> Continue
  ^.*BW\s+${SPEED}\s+Kbit -> Record

-> Continue stays in the state without consuming the line. -> Record writes the captured values as a record and resets. Five lines, and you've parsed multi-line interface stanzas. 

pythonimport textfsm

with open("my_template.textfsm") as tf:
    parser = textfsm.TextFSM(tf)

rows    = parser.ParseText(raw_output)
records = [dict(zip(parser.header, r)) for r in rows]

When TextFSM doesn't fit

Some platforms ship JSON or XML output natively — Junos with | display json, Arista with eAPI, NX-OS with | json. Use those first. Parsing json.loads(...) output is always more robust than parsing text. TextFSM is for the platforms and commands where structured output isn't an option.

The textfsm-related ecosystem is bigger than ntc-templates: genie from Cisco, ttp (Template Text Parser), and napalm's normalized getters all solve the same problem with different tradeoffs. ntc-templates is the easiest to start with; napalm is worth a look if you want a consistent API across vendors.

Pick a show command you parse with regex today. Look it up in ntc-templates (templates are named {platform}_{command}.textfsm). Replace your regex with parse_output. Delete the regex. Sleep better.
