tempfile: scratch space, done right

Files and directories that clean themselves up. Stop writing to /tmp/myfile.txt and forgetting to delete it.

23 Feb 2026 • ~2 min read • Python 3.0+
TL;DR

Use tempfile.TemporaryDirectory() as a context manager for a scratch directory that deletes on exit. Use tempfile.NamedTemporaryFile() for a single scratch file with a real path. Both are race-free, both clean up even on exceptions.

Hand-rolled scratch files leak. The script crashes, nobody deletes /tmp/output_42.json, and a year later the disk is full of forgotten artifacts. The tempfile module solves this with context managers that always clean up.

Scratch directories

pythonimport tempfile
from pathlib import Path

with tempfile.TemporaryDirectory() as tmp:
    workspace = Path(tmp)
    (workspace / "input.json").write_text(payload)
    subprocess.run(["some-tool", workspace / "input.json"])
    # do whatever — files go in `workspace`
# Directory and everything in it is gone here, even if we crashed.

That's the whole pattern. The directory exists for the duration of the with block; it's recursively deleted on exit, even on exception.

Single scratch files

For one file you'll hand off to another tool, use NamedTemporaryFile. The name attribute is the real path on disk. 

pythonwith tempfile.NamedTemporaryFile(mode="w", suffix=".json", delete=True) as f:
    json.dump(payload, f)
    f.flush()                    # important — let other tools see the bytes
    subprocess.run(["jq", ".", f.name])
# File is deleted here.

Two gotchas worth flagging:

  • On Windows, the file can't be opened by another process while NamedTemporaryFile still has it open. Pass delete=False, close it explicitly, then have the other tool read it. (As of Python 3.12, delete_on_close=False handles this cleanly.)
  • Always flush() before another process reads the file — the bytes might still be sitting in Python's buffer.

Just a path, no auto-open

Sometimes you want a unique temp filename without actually creating it — for instance, to pass as an output argument to another tool that creates the file itself. 

python# Generate a unique path; create + clean up yourself
tmp_path = Path(tempfile.mkdtemp()) / "output.bin"
try:
    subprocess.run(["some-tool", "--out", tmp_path])
    # now read tmp_path
finally:
    shutil.rmtree(tmp_path.parent)

Wrapping with try/finally is the manual version of what TemporaryDirectory gives you for free — prefer the context manager when it works.

Atomic writes: the related pattern

For "replace this file atomically" — write to a temp file in the same directory, then rename on top. This is the only way to avoid partial files if the program crashes mid-write. 

pythonimport os, tempfile
from pathlib import Path

def atomic_write(target: Path, data: str):
    # Same dir as target so rename is atomic (cross-FS rename can fail)
    fd, tmp = tempfile.mkstemp(dir=target.parent)
    try:
        with os.fdopen(fd, "w") as f:
            f.write(data)
        os.replace(tmp, target)            # atomic on POSIX and Windows
    except Exception:
        os.unlink(tmp)
        raise

Readers either see the old content or the new content — never half a file. Crucial for config files and any data file that another process might be watching.

By default, temp files go in /tmp on Linux, %TEMP% on Windows. Override the location for big scratch data with dir="/data/scratch" on any of these calls — sometimes /tmp is small or in-memory (tmpfs).

Find a script that does open("/tmp/something", "w") directly. Convert to NamedTemporaryFile or TemporaryDirectory. The cleanup becomes guaranteed; the filename becomes unique; you stop conflicting with parallel runs.
