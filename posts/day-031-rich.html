rich: terminal output that doesn't suck

Tables, colors, progress bars, and JSON pretty-printing in one library. The closest Python has to a "make my CLI look like a real product" button.

31 Jan 2026 • ~3 min read • Python 3.7+
TL;DR

from rich import print upgrades every print with markup and pretty-printing. rich.table.Table renders aligned, colored tables. rich.progress gives you a real progress bar wrapping any iterable. rich.console.Console().status() gives you a spinner. Each is one import.

For interactive scripts — CLIs, status dashboards, long-running ops tools — rich is the single highest-leverage dependency you can add. The default usage requires almost zero learning.

The two-line upgrade

pythonfrom rich import print

print("[bold green]✓[/] connected to [cyan]router-01[/]")
print({"interfaces": 200, "down": 3, "errors": ["timeout", "auth"]})

Square-bracket markup gives you BBCode-style colors and styles (bold, italic, underline, color names, hex codes). Native Python objects get pretty-printed with syntax highlighting instead of dumped flat.

Tables for structured data

pythonfrom rich.console import Console
from rich.table   import Table

t = Table(title="Interface status")
t.add_column("Interface", style="cyan")
t.add_column("IP",        style="magenta")
t.add_column("Status",    justify="right")

for iface in interfaces:
    color = "green" if iface["status"] == "up" else "red"
    t.add_row(iface["name"], iface["ip"],
              f"[{color}]{iface['status']}[/]")

Console().print(t)

Real columns, real alignment, real coloring — in roughly the time it would take to format the strings by hand and get them slightly wrong.

Progress bars wrap iterables

pythonfrom rich.progress import track
import time

for host in track(hosts, description="Probing devices"):
    probe(host)
    time.sleep(0.1)

That's the entire integration. track() wraps any iterable and gives you a progress bar with ETA, count, and percentage. For multi-stage progress, use the Progress class with multiple tasks.

Spinners for unknown-duration work

pythonfrom rich.console import Console

console = Console()
with console.status("[cyan]Connecting to vault…[/]"):
    creds = vault_login()

with console.status("[cyan]Pulling config…[/]", spinner="dots"):
    cfg = fetch_config(device)

Pretty-printing JSON and tracebacks

pythonfrom rich          import print_json
from rich.traceback import install

# Highlighted, indented JSON
print_json(data=my_dict)

# Replace the default traceback handler globally
install(show_locals=True)
# Now any unhandled exception renders with locals, syntax highlighting,
# and source context. Worth its weight in gold for debugging.

Logging integration

From Day 016: rich.logging.RichHandler is a drop-in replacement for the default stream handler — same logging API, much better-looking output. 

pythonimport logging
from rich.logging import RichHandler

logging.basicConfig(
    level="INFO",
    format="%(message)s",
    datefmt="[%X]",
    handlers=[RichHandler(rich_tracebacks=True)],
)
log = logging.getLogger("app")
log.info("Probe started for [cyan]router-01[/]")

rich detects whether stdout is a TTY and degrades gracefully when it isn't — markup gets stripped, colors removed, progress bars suppressed. Your scripts stay pipe-friendly without code changes. Override with force_terminal=True on the Console if you need colors in CI logs.

Take any script that prints status text. Add from rich import print at the top. Replace one block of formatted output with a Table. Wrap one loop in track(). The visual upgrade is dramatic; the diff is small.
